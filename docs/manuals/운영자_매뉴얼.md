# 운영자 매뉴얼

## 1. 이 문서는 누가 보면 되나

- 마스터
- 서무
- 운영 담당자
- 권한/설정, 구성원 관리, 운영실을 다루는 사람

쉽게 말하면:
- `직원들이 쓰는 시스템 뒤편을 관리하는 사람`을 위한 문서다.

## 2. 운영자가 하는 일

- 사용자 권한 설정
- 구성원 등록/수정
- 연차 총량/근무시간 관리
- 특별휴가 부여
- 운영실에서 대리 등록/수정/삭제
- 수동 휴일 설정
- 공휴일 동기화
- 접속 로그 확인 및 CSV 다운로드

## 3. 권한/설정 화면 이해

권한 탭은 기능별 권한을 사용자마다 체크하는 곳이다.

현재 그룹:
- 달력정보
- 승인권한
- 팀원 연차현황
- 전체 상황판
- 세부설정

### 3.1 세부설정

세부설정은 PC와 모바일 권한이 분리되어 있다.

세부설정 항목 예:
- 권한/설정 접근(PC/모바일)
- 구성원 관리(PC/모바일)
- 운영실 접근(PC/모바일)

주의:
- 운영실은 구성원 관리와 별도 권한이다.
- 전체 상황판도 PC/모바일 권한이 따로 있다.

## 4. 구성원 관리

구성원 관리에서는 다음을 다룬다.

- 신규 직원 등록
- 기존 직원 수정
- 이름/부서/직급/사번/전화번호/이메일 관리
- 총 연차 입력
- 분기별 근무시간 입력
- 특별휴가 부여

### 4.1 신규 등록 시 주의

- `Q1~Q4` 근무시간이 비어 있으면 저장이 막히도록 되어 있다.
- 이메일, 아이디, 기본 인적사항도 함께 확인한다.

### 4.2 총 연차 입력

- 입력은 `일수` 기준으로 하되, 내부 계산은 시간 기준으로 처리된다.
- 예를 들어 `19.5일`은 내부적으로 시간으로 계산된다.

## 5. 운영실

운영실은 운영자가 가장 자주 쓰는 화면이다.

쉽게 말하면:
- DB를 직접 여는 대신, 운영에 필요한 작업을 화면에서 처리하는 관리실이다.

### 5.1 다른 직원 연차 대리 등록

사용 목적:
- 누락된 기록 보정
- 관리자가 직접 등록해야 하는 경우

특징:
- 결재 대기 신청이 아니라 `운영자 직접 등록`이다.
- 저장 시 바로 반영된다.

### 5.2 다른 직원 연차 수정

사용 목적:
- 날짜, 종류, 시간차 시간, 사유 등이 잘못 들어갔을 때 보정

특징:
- 승인 절차가 아니라 `기록 정정`
- 상태는 유지하고 내용만 수정

### 5.3 다른 직원 연차 삭제

사용 목적:
- 잘못 입력된 기록 제거

주의:
- 삭제는 되돌리기 어려우므로 날짜와 직원을 다시 확인한다.

## 6. 수동 휴일 설정

수동 휴일 설정은 회사 휴일을 직접 넣는 기능이다.

예:
- 여름휴가
- 전사 워크숍
- 임시 휴무

구분:
- `자동 공휴일`은 정부/동기화 데이터
- `회사 고정 휴일`은 창립기념일, 노동절 등
- `수동 휴일`은 운영자가 직접 넣는 값

## 7. 공휴일 동기화

운영실에서는 대한민국 공휴일을 연도 범위로 동기화할 수 있다.

사용 예:
- `2026 ~ 2027`
- `2027 ~ 2028`

중요:
- 선택한 범위의 자동 공휴일만 갱신한다.
- 이전 연도의 수동 휴일은 지워지지 않는다.

쉽게 말하면:
- `선택한 연도 달력만 다시 갈아끼우는 방식`이다.

## 8. 접속 로그

운영실에서는 최근 접속 로그를 볼 수 있다.

할 수 있는 일:
- 최근 접속 로그 보기
- 전체 로그 CSV 다운로드

주의:
- CSV 다운로드는 전체 로그를 읽기 때문에 Firestore 읽기 사용량에 영향을 줄 수 있다.

## 9. 백업/복구 개념

운영자가 꼭 개발 스크립트를 직접 돌릴 필요는 없지만, 개념은 알아두는 것이 좋다.

### 9.1 GitHub

- 코드 보관함
- 화면/기능 설계도

### 9.2 Firestore

- 실제 운영 데이터 장부
- 직원, 신청, 휴일, 로그 데이터가 들어간다

### 9.3 백업

- 운영 장부 복사본을 뜨는 작업
- 큰 수정 전에는 백업이 안전하다

### 9.4 복구

- 백업 복사본을 다시 Firestore로 넣는 작업

## 10. 운영 시 주의사항

1. 권한은 PC/모바일이 따로일 수 있다.
2. 운영실 권한과 구성원 관리 권한은 별개다.
3. 수동 휴일과 자동 휴일을 혼동하지 않는다.
4. 대리 수정은 승인 절차가 아니라 기록 보정이다.
5. 중요한 변경 전에는 백업 여부를 확인한다.

## 11. 운영자용 자주 묻는 질문

### Q. 왜 운영실이 안 보이나

- 운영실 접근 권한이 없는 경우다.
- PC/모바일 권한이 따로일 수 있으니 둘 다 확인한다.

### Q. 왜 전체 상황판이 안 보이나

- 전체 상황판 접근 권한이 꺼져 있을 수 있다.
- 역시 PC와 모바일이 따로일 수 있다.

### Q. 수동 휴일을 넣었는데 안 보인다

- 날짜 형식, 저장 여부, 새로고침을 먼저 확인한다.
- 자동/수동 소스가 다를 수 있으니 휴일 목록도 같이 본다.

## 12. 한 줄 정리

- 권한은 `누가 무엇을 볼 수 있는지`
- 구성원 관리는 `사람 정보와 연차 기본값 관리`
- 운영실은 `실제 운영 보정과 관리`

즉 운영자는 `권한`, `사람`, `기록`, `휴일` 네 가지를 관리한다고 생각하면 된다.
