# 승인권자·대표 매뉴얼

## 1. 이 문서는 누가 보면 되나

- 파트리더
- 팀리더
- 대표
- 승인/반려를 처리해야 하는 사용자

쉽게 말하면:
- `내 신청`보다 `다른 사람 신청을 보고 결정하는 사람`을 위한 문서다.

## 2. 이 역할에서 주로 하는 일

- 결재 대기 목록 확인
- 승인
- 반려
- 팀원 연차현황 확인
- 전체 상황판 확인
- 자리배치도 확인

## 3. 결재 처리 방법

### 3.1 기본 흐름

1. 승인 화면 또는 결재 대기 영역으로 간다.
2. 대기 중인 신청을 확인한다.
3. `승인` 또는 `반려`를 선택한다.
4. 반려일 경우 사유를 입력한다.
5. 저장 후 상태가 바뀌었는지 확인한다.

### 3.2 승인할 때 볼 것

- 신청자 이름
- 부서
- 휴가 종류
- 날짜
- 시간차인 경우 시간 범위
- 사유

### 3.3 반려할 때 볼 것

- 날짜 오류
- 시간 범위 오류
- 사유 부족
- 권한/업무 일정 문제

주의:
- 반려 사유는 직원이 다시 신청할 때 참고하므로 짧더라도 분명하게 적는 것이 좋다.

## 4. 팀원 연차현황 보기

이 화면은 팀원들의 잔여 연차를 한눈에 보는 화면이다.

볼 수 있는 정보:
- 이름
- 직급
- 남은 연차
- 총 연차
- 사용 비율
- 특별휴가 요약

활용 예:
- 누가 연차 여유가 있는지 보기
- 특정 팀의 전체 사용량 보기
- 직원별 근무시간/기본정보 확인

## 5. 전체 상황판 보기

전체 상황판은 `오늘 누가 비었는지`, `조직 전체 일정이 어떤지` 빠르게 보는 화면이다.

### 5.1 볼 수 있는 것

- 날짜별 부재 요약
- 연차
- 반차
- 시간차(퇴근)
- 시간차(외출)
- 장기 부재
- 대기

### 5.2 전체 상황판을 언제 쓰나

- 대표가 조직 전체 상황을 볼 때
- 팀장이 특정 날짜 인원 공백을 볼 때
- 회의/업무 조율 전에 당일 인원 상태를 볼 때

### 5.3 자리배치도

- 자리배치도는 `누가 실제로 자리에 있는지` 더 직관적으로 보는 화면이다.
- 오늘 날짜는 현재 시각 기준으로 실제 부재 여부가 반영된다.

쉽게 말하면:
- 달력형 전체 상황판 = 일정표
- 자리배치도 = 지금 비어 있는 자리 표시판

## 6. 모바일에서 보는 방법

- 모바일에서는 상단 프로필 메뉴에서 기능을 연다.
- 전체 상황판은 권한이 있는 경우만 보인다.
- 모바일 달력은 날짜를 눌러 상세를 보는 방식이다.
- 상황판과 달력은 좁은 화면에 맞게 요약 중심으로 보인다.

## 7. 자주 묻는 질문

### Q. 승인했는데 반영이 안 된 것처럼 보인다

- 새로고침 후 다시 확인한다.
- 상태가 `승인 완료`로 바뀌는지 본다.

### Q. 시간차는 왜 따로 보나

- 시간차는 하루 전체 부재가 아니라 특정 시간만 비는 경우라서, 시간 범위가 중요하다.

### Q. 대표는 어떤 화면을 제일 많이 보나

- `전체 상황판`
- `자리배치도`
- 필요한 경우 `팀원 연차현황`

## 8. 승인권자용 체크 포인트

1. 결재 대기 목록을 너무 오래 쌓아두지 않는다.
2. 반려할 때는 짧아도 이유를 남긴다.
3. 팀 일정이 몰린 날짜는 상황판으로 먼저 확인한다.
4. 시간차는 시간 범위를 꼭 본다.

## 9. 한 줄 정리

- 승인권자는 `결정`
- 대표는 `전체 파악`

즉 이 문서의 핵심은 `누가 쉬는지 보고, 필요한 결정을 빠르게 내리는 방법`이다.
