# 매뉴얼 작성 계획

## 1. 목적

- 현재 운영 중인 `NCORE 연차관리 시스템`을 처음 쓰는 사람도 이해할 수 있게 역할별 매뉴얼을 만든다.
- 개발 문서가 아니라, 실제 사용자가 `어디서 무엇을 누르면 되는지` 알 수 있는 안내 문서를 만든다.
- 시연, 인수인계, 신규 사용자 교육에 바로 사용할 수 있는 수준으로 정리한다.

쉽게 말하면:
- 건물은 이미 지어졌고,
- 이번 문서는 `층별 안내도`, `직원용 안내문`, `관리실 작업 설명서`를 붙이는 작업이다.

## 2. 작성 범위

### 2.1 사용자 매뉴얼

- 일반 직원 기준
- 로그인
- 홈 화면 이해
- 연차/반차/시간차/특별휴가 신청
- 최근 신청 내역 확인
- 캘린더 사용
- 모바일 사용 흐름

### 2.2 승인권자/대표 매뉴얼

- 승인권자, 팀장, 대표 기준
- 승인/반려/취소 처리
- 팀원 연차현황 확인
- 전체 상황판 확인
- 자리배치도 확인

### 2.3 운영자 매뉴얼

- 마스터/서무/운영 담당 기준
- 권한 설정
- 구성원 관리
- 운영실
- 휴일 관리
- 공휴일 동기화
- 로그 CSV 다운로드
- 백업/복구 개념

### 2.4 빠른 시작 체크리스트

- 첫 로그인 후 확인할 것
- 역할별 첫 작업
- 운영 전 점검 항목

## 3. 작성 원칙

1. 개발자 용어보다 사용자 용어를 쓴다.
2. 기능 설명보다 `실제 사용 순서`를 먼저 적는다.
3. 한 문서에는 한 역할만 넣는다.
4. 실수하기 쉬운 부분은 `주의`로 따로 뺀다.
5. PC와 모바일 차이가 큰 경우는 따로 적는다.
