# 작업원칙: 즉시 웹 반영

## 원칙

앞으로는 수정 작업을 할 때 아래 원칙을 기본으로 한다.

1. 코드 수정
2. 바로 운영 웹 반영

쉽게 비유하면:
- 작업실에서 오래 검사한 뒤 진열하는 방식이 아니라,
- 수정이 끝나면 바로 매장 진열대로 보내는 방식이다.

- 기준 폴더에서 바로 수정

### 2단계. 운영 웹 반영

```powershell
cd D:\MYDOC\Desktop\vscode\Web\firebase\ncore_web\ncore-main
cmd /c npx firebase-tools deploy --only functions,hosting --project ncore-vacation-system
```

## 적용 이유

이 원칙을 쓰는 이유:

1. 지금은 운영 반영 속도를 우선하기로 함
2. 이전처럼 수정 후 바로 웹에서 확인하는 흐름으로 돌아감
3. 로컬 확인은 필요할 때만 보조적으로 사용

## 보조 확인

- 필요하면 로컬 확인을 할 수는 있음
- 하지만 기본 원칙은 `로컬 완성 후 반영`이 아니라 `수정 후 바로 반영`이다

즉:
- 로컬은 선택사항
- 운영 웹 반영이 기본
