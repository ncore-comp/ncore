## 프런트 구현 계획

### 1. 운영실 보고 블록에 주간 안내 박스 추가
- 위치:
  - `req-report-preview` 아래
- 표시 내용:
  - 주간 기준 기간
  - 기본근로시간
  - 기존 잔업/특근
  - 이번 신청
  - 반영 후 예정 총 근로시간
  - 잔여 가능시간

### 2. 관리자 대리등록용 계산 함수 재사용
- 이미 있는 `getWeeklyWorkLimitSummary()` 재사용
- 단, `user`는 현재 로그인 사용자가 아니라
  - `app.getRequestModalTargetUser()`
를 기준으로 계산

### 3. 특근은 운영실에서도 `종일`
- 운영실 대리등록에서도 특근은 시간 입력 없이 `종일`
- 시간 입력 UI는 숨김 또는 비활성

### 4. 저장 차단
- `submitRequest()`에서
  - `isAdminProxy && isReportRequest`
인 경우도 동일하게 `52시간 초과` 차단

## 서버 구현 계획

### 1. 검증 함수 추가
- 서버에서 계산할 항목:
  - 해당 주 기본근로시간
  - 기존 잔업/특근
  - 현재 저장하려는 잔업/특근

### 2. 기본근로시간 계산
- 사용자 `workQ1 ~ workQ4`
- `종료 - 시작 - 1시간 휴게`
- 평일(월~금) + 공휴일 제외

### 3. 특근 환산
- 특근 1건 = 해당 날짜 일일 기본근로시간

### 4. 제외 규칙
- `cancelled`, `rejected` 제외
- 잔업/특근에서 `settlementStatus = rejected`인 건도 제외
- 수정 중인 현재 문서는 자기 자신 제외

### 5. 실패 응답
- 예시 코드:
  - `WORK_LIMIT_EXCEEDED`
- 포함 정보:
  - `weekStart`
  - `weekEnd`
  - `basicHours`
  - `existingReportHours`
  - `currentRequestHours`
  - `totalHours`
