# 로그 읽기 절감 1차 계획
# Log Read Reduction Phase 1 Plan

## 목적
## Goal
- 운영실 진입 시 `접속 로그`, `보안 로그`를 자동으로 읽지 않도록 바꾼다.
- Stop auto-loading `access logs` and `security logs` when entering Admin Ops.
- 로그는 필요할 때만 최근 20건씩 수동으로 읽도록 바꾼다.
- Load only the latest 20 log entries on demand.
- 작은 수정으로 Firestore 읽기 수를 즉시 줄인다.
- Reduce Firestore reads immediately with the smallest safe change.

## 현재 문제
## Current Problem
- 운영실 진입만 해도 최근 로그를 바로 읽고 있다.
- Entering Admin Ops immediately reads recent logs.
- 화면에는 최근 20건만 보여도, 진입 시마다 읽기 비용이 발생한다.
- Even though only 20 entries are shown, each entry into the screen incurs read cost.
- `accessLogs` 컬렉션이 커질수록 운영실 진입 비용도 계속 커진다.
- As `accessLogs` grows, the Admin Ops entry cost keeps increasing.

## 이번 단계 범위
## Scope For This Phase
1. 운영실 첫 진입 시 로그 자동 로드 제거
1. Remove automatic log loading on first Admin Ops entry
2. `접속 로그 불러오기`, `보안 로그 불러오기` 버튼 추가
2. Add `Load Access Logs` and `Load Security Logs` buttons
3. 버튼을 눌렀을 때만 최근 20건 조회
3. Fetch only the latest 20 records when the button is clicked
4. 이미 읽은 뒤에는 `새로고침` 버튼으로 다시 조회
4. After loading once, change the button to `Refresh`
5. 휴일/운영실 기본 정보는 계속 정상 로드
5. Keep holiday and Admin Ops base data loading as before

## 이번 단계에서 하지 않을 것
## Out Of Scope For This Phase
- CSV 다운로드 범위 제한
- CSV download range limiting
- 로그 더보기 / 페이지네이션
- Log pagination / load more
- 오래된 로그 아카이브
- Archiving old logs
- `requests`, `users`, `holidays` 읽기 구조 최적화
- Optimization of `requests`, `users`, and `holidays` loading

## 구현 방향
## Implementation Direction

### backend
### backend
- `load_admin_ops_data`가 아래 옵션을 받도록 확장
- Extend `load_admin_ops_data` to accept:
  - `includeAccessLogs`
  - `includeSecurityLogs`
- 옵션이 `false`이면 해당 로그 조회를 생략
- If an option is `false`, skip reading that log collection
- 휴일과 운영실 기본 정보는 기존처럼 반환
- Continue returning holiday/admin base data as before

### frontend
### frontend
- 상태값 추가
- Add state flags
  - `adminOpsAccessLogsLoaded`
  - `adminOpsSecurityLogsLoaded`
- 운영실 첫 진입
- On first Admin Ops entry
  - 로그는 읽지 않고 운영 정보와 휴일 요약만 읽는다
  - Do not read logs; only load admin metadata and holiday summary
  - 로그 패널은 `미로드` 상태로 렌더링한다
  - Render log panels in a `not loaded` state
- 버튼 클릭 시
- On button click
  - 해당 로그만 최근 20건 조회
  - Fetch only that log type, latest 20 items
  - 이후 버튼은 `새로고침`으로 바뀐다
  - Then switch the button label to `Refresh`

## 기대 효과
## Expected Effect
- 운영실 진입만으로 발생하던 불필요한 로그 read 제거
- Remove unnecessary log reads on Admin Ops entry
- “조금만 확인했는데 read가 급증”하는 현상 완화
- Reduce the “I barely used it, why did reads spike?” problem
- 구조 변경 없이 빠르게 적용 가능
- Safe and fast to apply without major architectural changes
