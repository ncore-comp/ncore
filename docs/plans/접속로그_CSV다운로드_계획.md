# 접속로그 CSV 다운로드 계획

## 목적
- 운영실에서 접속 로그를 `항상 전체 기준`으로 CSV 파일로 내려받을 수 있게 한다.

## 쉬운 비유
- 지금 운영실 로그 패널은 `최근 기록만 보이는 모니터`다.
- 여기에 `출입 장부 전체를 인쇄하는 버튼`을 추가하는 작업이다.

## 요구사항
1. 버튼은 운영실 `최근 접속 로그` 카드에 둔다.
2. 다운로드 대상은 `현재 화면 20건`이 아니라 `전체 accessLogs`다.
3. CSV는 최신순으로 정렬한다.
4. 한글 깨짐 방지를 위해 `UTF-8 BOM`을 포함한다.

## 구현 방향

### 1. 서버
- 새 액션 추가
  - 예: `download_access_logs_csv`
- 서버가 `accessLogs` 전체를 읽는다.
- `sortTimestamp` 기준 최신순으로 정렬한다.
- CSV 문자열을 만들어 응답으로 보낸다.

### 2. 프런트
- 운영실 `최근 접속 로그` 카드 상단에 `CSV 다운로드` 버튼 추가
- 버튼 클릭 시 새 액션 호출
- Blob으로 받아 `.csv` 파일로 저장

## CSV 컬럼 제안
1. timestamp
2. userName
3. userId
4. type
5. ip
6. detail
7. sortTimestamp

## 파일명 제안
- `accessLogs_YYYY-MM-DD_HHmmss.csv`

## 권한
- 운영실 접근 가능한 관리자만 사용 가능

## 완료 기준
1. 운영실에서 버튼이 보인다
2. 버튼 클릭 시 CSV 파일이 다운로드된다
3. CSV에는 `accessLogs` 전체가 담긴다
4. 최신순 정렬이 맞다

## 검증 계획
1. 운영실 버튼 표시 확인
2. CSV 파일 다운로드 확인
3. 파일 첫 행 컬럼 확인
4. 행 수가 최근 20건보다 많거나 전체 기준인 것 확인
5. 최신 로그가 상단에 오는지 확인
