# 공휴일 동기화 범위 계획

## 목적
- 운영실에서 공휴일 동기화 실행 시, 관리자가 `시작연도 ~ 종료연도`를 직접 지정할 수 있게 한다.

## 쉬운 비유
- 지금 버튼은 `창고 달력 전체를 한 번에 갈아끼우는 스위치`다.
- 바꾸려는 방식은 `몇 년도 서랍만 새로고칠지 고르는 다이얼`을 버튼 옆에 다는 것이다.

## 현재 상태
- 운영실의 `공휴일 동기화` 버튼은 고정 범위로만 실행된다.
- 서버는 이미 `startYear`, `endYear`를 받을 수 있다.
- 즉 백엔드는 준비돼 있고, 관리자가 범위를 직접 고르는 UI가 없는 상태다.

## 변경 방향
1. 운영실 `수동 휴일 설정` 카드에 `시작연도`, `종료연도` 입력칸 추가
2. 기본값
   - 시작연도: 현재 연도
   - 종료연도: 현재 연도 + 1
3. 버튼 클릭 시 해당 범위를 서버로 전달

## 검증 규칙
1. 시작연도와 종료연도는 숫자여야 한다
2. 시작연도 > 종료연도면 실행 차단
3. 범위가 너무 크면 확인창 한 번 더 띄운다
   - 예: 3년 이상

## 유지 원칙
- 선택 범위에 해당하는 `auto`, `company_fixed`만 갱신
- 그 외 연도는 그대로 유지
- `manual` 수동 휴일은 그대로 유지
