## 1차 수정 대상

### 1. 권한 탭 `잔업/특근` 무반응
- 원인:
  - 버튼은 있는데 `setPermissionColumnOpen()` 허용 목록에 `workreport`가 빠져 있음
- 조치:
  - `workreport`를 허용 목록에 추가

### 2. `설정 테이블`과 `잔업/특근 설정`의 중복 토글
- 원인:
  - `global-overtime-toggle`, `global-holiday-work-toggle`가 두 탭에 동시에 존재
  - 동일 DOM id 중복 사용
- 조치:
  - 잔업/특근 토글은 `잔업/특근 설정` 탭에만 남기고, `설정 테이블`에서는 제거

### 3. 수정 모달이 뒤로 깔리는 문제
- 원인:
  - `월별 잔업/특근 현황` 모달을 닫지 않고 `잔업/특근 수정` 모달을 추가로 열고 있음
- 조치:
  - 수정 버튼 클릭 시 현황 모달을 먼저 닫고 수정 모달로 전환

## 2차 수정 대상

### 4. 특근은 하루 기준
- 요구:
  - 특근은 시작시간/보고시간이 필요 없음
- 조치:
  - 특근 선택 시 시간 입력 블록 숨김
  - 저장값은 `종일`, `reportedHours = 0`, `holiday_work` 기준으로 정리
  - 상세/현황/메일에서도 특근은 시간 대신 `종일` 또는 `1일` 기준으로 표시

### 5. 메일초안 제거
- 요구:
  - `메일초안` 버튼 제거
  - `승인 요청` 시 자동 합산된 메일을 바로 작성
- 조치:
  - `메일초안` 버튼 제거
  - `승인 요청` 시 메일 수신자 설정을 읽어 자동으로 제목/본문을 만들고 메일 앱을 열기
  - 메일 내용은 상세 행 전부가 아니라 정산기간, 총 잔업시간, 총 특근일수, 웹 상세 확인 안내 중심으로 단순화
