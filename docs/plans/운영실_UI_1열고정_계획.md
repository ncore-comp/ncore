# 운영실 UI 1열 고정 계획

## 1. 목적

- 운영실 UI가 브라우저 너비에 따라 1열 / 2열로 바뀌지 않게 한다.
- 사용자가 선호한 `브라우저 가로 절반폭` 상태의 UI를 기준형으로 고정한다.

쉽게 비유하면:
- 지금은 책상을 넓은 방에 놓으면 옆으로 두 줄 배치되고, 좁은 방에선 세로 한 줄로 놓인다.
- 이번 작업은 책상 배치를 항상 `세로 한 줄`로 고정하는 것이다.

## 2. 현재 상태

운영실 본문은 현재 아래 구조다.

- 기본: 1열
- `xl` 이상: 2열

즉:
- 좁은 브라우저 = 사용자가 좋아한 1열 UI
- 넓은 브라우저 = 현재 바뀌는 2열 UI

## 3. 목표 상태

브라우저 크기와 상관없이 운영실 본문은 항상 아래 순서로 세로 배치된다.

1. 운영실 요약 카드
2. 달력 기반 대리 등록/삭제
3. 수동 휴일 설정
4. 최근 접속 로그

## 4. 구현 방향

### 4-1. 레이아웃 고정

현재 운영실 본문 grid에서

- `xl:grid-cols-[minmax(0,1.5fr)_420px]`

를 제거하고

- `grid grid-cols-1`

만 유지한다.

### 4-2. 시각적 흐름 유지

- 상단 운영실 요약 카드는 그대로 유지
- 하단 본문은 항상 세로 스택
- 최근 접속 로그 패널도 하단으로 내려와 고정

### 4-3. 기능 구조는 그대로 유지

이번 작업은 `배치 고정`만 한다.

즉:
- 접속 로그 기능
- 대리 등록/수정/삭제
- 수동 휴일 설정
- 공휴일 동기화

기능 로직은 건드리지 않는다.

## 5. 기대 효과

1. 브라우저 크기에 따라 운영실 레이아웃이 흔들리지 않는다.
2. 시연/설명할 때 화면이 더 일관된다.
3. 사용자가 선호한 가독성 좋은 형태를 항상 유지할 수 있다.

## 6. 검증 계획

1. 넓은 브라우저에서 운영실 진입
   - 1열 세로 배치 유지 확인
2. 좁은 브라우저에서 운영실 진입
   - 기존 1열 배치 유지 확인
3. 기능 영역 순서 확인
   - 운영실 요약
   - 대리 등록/삭제
   - 수동 휴일 설정
   - 최근 접속 로그

## 7. 완료 기준

아래가 되면 완료로 본다.

1. 브라우저 크기와 상관없이 운영실이 항상 1열로 보인다.
2. 사용자가 선호한 두 번째 이미지와 같은 흐름으로 유지된다.
3. 기능 동작에는 영향이 없다.
