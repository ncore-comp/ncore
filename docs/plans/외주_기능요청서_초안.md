# NCORE 연차관리 시스템 외주 기능요청서 초안

## 1. 문서 목적

본 문서는 현재 운영 중인 `NCORE 연차관리 시스템`과 **동일하거나 매우 유사한 수준의 기능**을 외주 개발사에 요청하기 위한 기능 명세 초안입니다.

견적 목적:
- 현재 운영 중인 시스템을 기준으로 재구축 비용을 산정하기 위함
- 페이지 수가 아니라 **실제 업무 기능 단위**로 범위를 전달하기 위함

운영 주소:
- [https://ncore.web.app](https://ncore.web.app)

---

## 2. 프로젝트 개요

### 2.1 시스템 성격

- 사내 직원용 연차/휴가 관리 시스템
- 관리자/대표가 조직 전체의 연차 현황을 실시간으로 확인할 수 있는 내부 업무 시스템
- 일반 홈페이지가 아니라 **업무 프로세스용 웹 애플리케이션**

### 2.2 주요 사용 대상

- 일반 직원
- 승인권자(팀리더, 파트리더, 대표)
- 관리자(마스터, 서무/운영 담당)

### 2.3 지원 환경

- PC 웹
- 모바일 웹
- 반응형 대응 필요

## 6. 연차 신청 기능

### 6.1 신청 종류

- 연차
- 반차(오전)
- 반차(오후)
- 시간차(퇴근)
- 시간차(외출)
- 특별휴가(관리자가 등록한 종류 기준)

### 6.2 신청 입력 항목

- 종류
- 날짜(단일/기간)
- 시간 범위(시간차의 경우)
- 사유(선택값)
- 상세사유(필요한 경우)

### 6.3 과거 신청 처리

- 이전 날짜 신청 허용 기능
- 이전 날짜 또는 이미 지난 시간의 시간차 신청 시 **상세사유 필수**

### 6.4 시간차 처리

- 시간차(퇴근): 사용 시간에 따라 퇴근시간 자동 계산
- 시간차(외출): 시작 시각 + 사용시간 기반 처리
- 분기별 근무시간 기준 반영

### 6.5 휴일/특별휴가 처리

- 일반 휴일에 대한 신청 제한 또는 예외 허용
- 특별휴가 종류별 차감 방식/허용 방식 반영

## 10. 관리자 운영실

운영실은 일반 권한/설정과 별도 성격의 운영 관리 화면이어야 함.

### 10.1 달력 기반 대리 등록 / 수정 / 삭제

- 관리자가 다른 직원의 연차를 직접 등록
- 기존 신청 이력 직접 수정
- 기존 신청 이력 직접 삭제
- 승인 절차 없이 **관리자 직접 정정** 방식

### 10.2 접속 로그

- 최근 접속 로그 표시
- CSV 전체 다운로드

### 10.3 수동 휴일 설정

- 회사 휴일 등록/수정/삭제
- 공공데이터 공휴일과 별도 관리

### 10.4 공휴일 동기화

- 대한민국 공휴일 자동 동기화
- 연도 범위 지정
- 회사 고정 휴일(예: 창립기념일, 노동절) 포함 관리

### 10.5 접근 권한

- 운영실 접근도 PC/모바일 권한으로 따로 분리
- 운영실 권한이 있는 사용자는 운영실만 진입 가능해야 함

## 21. 견적 산정 시 주의

이 시스템은 단순 페이지 수가 아니라 아래 난이도가 포함됨.

- 권한 분기
- 조직 단위 승인 로직
- 관리자 운영실
- 연차/시간차/특별휴가 계산
- 공휴일 자동/수동 병합
- 모바일 전용 UI 보정
- 전체 상황판/자리배치도

따라서 `홈페이지 제작 단가` 기준이 아니라 `내부 업무 시스템` 기준으로 산정해야 함.
