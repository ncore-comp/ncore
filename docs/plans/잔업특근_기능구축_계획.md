# 잔업/특근 기능 구축 계획

기록 시각: 2026-03-30 14:18 KST

## 목표
- `권한/설정 > 설정 테이블`에 `잔업 신청 기능`, `특근 신청 기능` 토글을 추가한다.
- 토글이 켜지면 메인 화면 `연차 신청` 버튼 왼쪽에 `잔업 신청`, `특근 신청` 버튼이 보이게 한다.
- 잔업/특근은 기존 연차 신청 흐름을 재사용하되, `승인 없는 보고형 데이터`로 처리한다.
- 버튼을 누르면 월별 `잔업/특근 현황판`을 열고, 그 안에서 보고를 추가할 수 있게 한다.
- `특별휴가 종류`는 기본값을 `닫힘`으로 두고 `펼치기/닫기`를 지원한다.

## 구현 방향
### 1. 추천 방향
- 기존 `requests` 저장 구조를 확장한다.
- 새 타입:
  - `잔업`
  - `특근`
- 새 상태:
  - `reported`
- 새 필드:
  - `reportCategory`
  - `workDetail`
  - `requestDept`
  - `note`
  - `requestedStartAt`
  - `requestedEndAt`
  - `reportedHours`

### 2. 잔업 규칙
- 직원의 분기별 저장된 퇴근 시간 기준으로 시작 시간을 자동 설정한다.
- 1시간 단위로 보고 시간을 선택한다.
- 승인 대기 없이 `보고 완료(reported)` 상태로 저장한다.

### 3. 특근 규칙
- 사용자가 시작 시간과 보고 시간을 직접 입력한다.
- 1시간 단위로 계산한다.
- 승인 대기 없이 `보고 완료(reported)` 상태로 저장한다.

## UI 계획
### 설정 화면
- 신규 토글 2개 추가
  - `잔업 신청 기능`
  - `특근 신청 기능`
- `특별휴가 종류` 영역을 기본 닫힘으로 변경

### 메인 화면
- 신규 버튼 2개 추가
  - `잔업 신청`
  - `특근 신청`
- 클릭 시 바로 신청창이 아니라 `월별 잔업/특근 현황판`을 연다.

### 월별 현황판
- 컬럼
  - 직원명
  - 일자
  - 시간
  - 구분
  - 사유
  - 요청 부서
  - 상세 보기
- 상단 버튼
  - 이전달
  - 다음달
  - 보고 추가

### 보고 모달
- 입력 항목
  - 날짜
  - 시작 시간
  - 보고 시간
  - 사유
  - 업무내용
  - 요청 부서
  - 비고

## 제외 규칙
- 잔업/특근은 연차 차감에서 제외
- 승인 대기/승인함/취소 요청함에서 제외
- 달력 부재 표시와 전체 상황판 부재 계산에서 제외

## 구현 순서
1. 설정 토글 추가
2. 메인 버튼 노출 추가
3. 요청 저장 구조 확장
4. 잔업/특근 보고 모달 추가
5. 월별 현황판 추가
6. 특별휴가 종류 접기/펼치기 추가
7. 운영판 검증 및 문서 정리
