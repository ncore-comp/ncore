## 한 줄 요약

- 잔업/특근 신청 저장 전에 `이번 주 총 예정 근로시간`을 계산해 `52시간`을 넘으면 저장을 차단합니다.
- 일반적인 근로시간제 기준으로 `법정근로 40시간 + 연장근로 12시간 = 주 최대 52시간`을 시스템 규칙으로 반영합니다.

쉽게 비유하면:
- 지금은 잔업/특근 신청서가 바로 접수되는 상태
- 앞으로는 접수 직전에 `주간 근무시간 계산기`가 한 번 돌고
- `52시간 초과`가 나오면 차단하는 구조입니다.

## 적용 전제

- 일반적인 근로시간제 기준으로 적용
- 탄력근로제, 선택근로제, 특별연장근로 인가 같은 예외 제도는 현재 사용하지 않는 것으로 간주
- 주 계산 기준은 `휴일 포함 7일`

## 목적

1. 잔업/특근 신청 단계에서 법정 한도를 넘는 입력을 미리 차단
2. 승인권자가 나중에 수작업으로 근무시간 합계를 다시 계산하지 않도록 보조
3. 직원/팀장/운영실이 동일한 기준으로 입력하도록 통일

## 계산 대상

### 포함
- 해당 주의 기본 소정근로시간
- 해당 주의 잔업(`overtime`)
- 해당 주의 특근(`holiday_work`)

### 제외
- 연차/반차/시간차 자체는 근로시간이 아니므로 주52시간 합산 대상에서 제외
- 취소/반려/삭제된 요청

## 기본 소정근로시간 계산 기준

### 1. 분기별 근무시간 사용
- 사용자별 `workQ1 ~ workQ4` 값을 사용
- 예: `07:00 ~ 16:00`이면 휴게 제외 실제 근로시간은 `8시간`으로 계산

### 2. 휴게시간 처리
- 현재 시스템 설명과 법 기준을 맞추기 위해
- `09:00 ~ 18:00`, `07:00 ~ 16:00` 같은 표기는 `9시간 체류 / 8시간 근로`로 해석
- 즉 기본근로시간 계산은 `종료 - 시작 - 1시간 휴게`

### 3. 특근
- 특근은 현재 시스템 정책상 `종일` 단위
- 계산상 특근 1건은 `해당 직원의 그 날짜 기본근로시간`만큼 반영
- 예:
  - 평일 기본근로시간이 8시간인 직원은 특근 1건 = 8시간

## 주간 계산 기준

### 1. 주 시작일
- `월요일 시작 ~ 일요일 종료`

### 2. 신청 저장 시 확인 범위
- 신청 대상 날짜가 포함된 주간 범위를 계산
- 그 주의:
  - 기본 소정근로시간
  - 기존 저장된 잔업/특근
  - 이번에 새로 입력한 잔업/특근
를 합산

### 3. 합산 공식
- `주간 예정 총 근로시간 = 주간 기본근로시간 + 기존 잔업시간 + 기존 특근환산시간 + 신규 입력시간`

## 잔업 규칙

### 저장 시
- 잔업 1건 저장 전에 해당 주 총합 계산
- `52시간 초과`이면 저장 차단

### 표시
- 잔업 신청 모달 하단에 안내 문구 표시
  - `이번 주 예정 총 근로시간: 49시간`
  - `저장 시 52시간을 초과하면 신청할 수 없습니다`

## 특근 규칙

### 저장 시
- 특근은 하루 단위이므로 시작시간/보고시간 없이 계산
- 해당 주 총합 계산 후 `52시간 초과`이면 저장 차단

### 표시
- 특근 신청 시:
  - `특근 1건은 종일 기준으로 계산됩니다`
  - `이번 주 예정 총 근로시간: n시간`

## 사용자 경고 문구

### 차단 문구 추천
- `주간 예정 근로시간이 52시간을 초과하여 저장할 수 없습니다.`

### 상세 문구 추천
- `이번 주 예정 총 근로시간: 54시간`
- `기본근로 40시간 + 잔업/특근 14시간으로 계산되었습니다.`

## 적용 위치

### 프런트 1차
- `잔업/특근 신청` 모달 저장 직전 계산
- 즉시 사용자에게 안내

### 서버 2차
- 클라이언트 우회 방지용으로 Cloud Functions 저장 단계에서도 같은 검증 추가
- 최종적으로는 `프런트 안내 + 서버 차단` 2중 구조가 안전

## 운영실/관리자 대리등록

- 운영실에서 직원 대신 잔업/특근을 등록할 때도 동일 규칙 적용
- 관리자라고 해서 주52시간 규칙을 우회하지 않음

## 예외 처리

- 실제 운영에서 예외 제도를 사용할 계획이 생기면
  - 예외 허용 플래그
  - 예외 사유 입력
  - 보안 로그 기록
를 별도로 설계해야 함
- 현재는 예외 없이 일괄 차단

## 구현 순서 제안

1. 분기 근무시간 문자열을 `일일 실제 근로시간`으로 변환하는 유틸 추가
2. 특정 날짜 기준 `해당 주 범위` 계산 유틸 추가
3. 주간 기본근로 + 잔업/특근 합산 함수 추가
4. 잔업 신청 저장 전 차단
5. 특근 신청 저장 전 차단
6. 운영실 대리등록에도 동일 적용
7. 서버 검증 추가

- 특근 저장 시 주52시간 초과면 차단
- 직원/팀장/운영실 모두 같은 기준 적용
