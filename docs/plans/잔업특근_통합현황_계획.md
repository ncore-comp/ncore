# 잔업특근 통합현황 계획

## 작업 일시
- 2026-04-01

## 작업 배경
- 현재 `월별 잔업/특근 현황`은 정산월 기준 통합 테이블로 바뀌었지만, 모든 사용자가 같은 형태의 장부를 보고 있습니다.
- 사용자는 여기서 한 단계 더 나아가, `개인용 현황`과 `관리자용 현황`을 나눠야 한다고 요청했습니다.
- 핵심 요구는 아래와 같습니다.
  - 일반 직원은 **본인 기록만** 봐야 한다.
  - 관리자는 **직원별 총 잔업시간 / 총 특근일수**를 먼저 보고,
  - `상세`를 누르면 그 직원이 **언제 했는지 상세내역**을 볼 수 있어야 한다.

## 목표
- 잔업/특근 현황을 역할 기준으로 두 가지 형태로 분리
  - `개인용 현황`
  - `관리자용 현황`
- 정산 기준은 그대로 유지
  - `매월 16일 ~ 다음달 15일`

## 정산 기준
- 예:
  - `2026년 3월 정산 = 2026-02-16 ~ 2026-03-15`
- `이전달`, `다음달` 이동도 정산월 단위로 유지

## 권장 구현 방향
### 1. 개인용 현황
- 일반 직원이 `잔업/특근신청` 버튼을 눌렀을 때 보는 기본 장부
- 본인 기록만 표시
- 상단 total:
  - `내 총 잔업시간`
  - `내 총 특근일수`
- 하단 테이블:
  - `일자`
  - `시간`
  - `구분`
  - `사유`
  - `상세`
- `직원명`, `요청 부서`는 숨김

### 2. 관리자용 현황
- 마스터 / 운영권한자 / 대표 등 넓은 열람이 필요한 사용자가 보는 장부
- 첫 화면은 `직원별 집계표`
- 상단 total:
  - 정산월 전체 `총 잔업시간`
  - 정산월 전체 `총 특근일수`
- 집계 테이블:
  - `직원명`
  - `총 잔업시간`
  - `총 특근일수`
  - `상세`
- `상세` 클릭 시 해당 직원의 정산월 상세내역 모달 오픈

## 관리자 상세 모달 구조
- 제목:
  - `홍길동 · 2026년 3월 정산 잔업/특근 상세`
- 보조 문구:
  - `집계 기준: 2026-02-16 ~ 2026-03-15`
- 테이블 컬럼:
  - `일자`
  - `시간`
  - `구분`
  - `사유`
  - `요청 부서`
  - `비고`

## 데이터 처리 기준
### 공통
- 대상은 `reported` 상태의 잔업/특근 보고만 사용
- `reportCategory`
  - `overtime` = 잔업
  - `holiday_work` = 특근

### 개인용 total
- `총 잔업시간`
  - 본인 기준 정산월 내 `reportedHours` 합산
- `총 특근일수`
  - 본인 기준 정산월 내 `특근 날짜 수`
  - 같은 날짜 특근이 여러 건이어도 1일로 집계

### 관리자용 total
- `총 잔업시간`
  - 전체 직원 기준 정산월 내 `reportedHours` 합산
- `총 특근일수`
  - 전체 직원 기준 정산월 내 `사용자 + 날짜` unique 기준 집계

### 관리자 집계표
- 한 직원당 1행
- `총 잔업시간`
  - 그 직원의 정산월 잔업 시간 총합
- `총 특근일수`
  - 그 직원의 정산월 특근 날짜 수

## 권한 기준
- 일반 직원:
  - 본인 현황만 열람
- 관리자/운영자/대표/마스터:
  - 관리자용 집계표 열람
- 세부 열람 범위는 기존 권한 체계와 맞춰 추가 검토
  - 예: 운영실 권한자, 전체상황판 권한자, 마스터의 차이

## UI/UX 제안
### 개인용
- 지금의 통합 테이블 구조를 유지하되 더 단순하게
- `내 장부` 느낌으로 가볍게 구성

### 관리자용
- `전체 원본 목록`을 바로 보여주지 말고
- 먼저 `직원별 집계표`
- 필요할 때만 `상세` 클릭
- 즉, `회사 집계표 -> 개인 상세 원장` 구조

## 구현 범위
- 현황 모달 진입 시 사용자 역할 판별
- 개인용 렌더링 함수 추가
- 관리자용 집계표 렌더링 함수 추가
- 관리자 상세 모달 추가
- 정산월 total 계산을 개인용 / 관리자용으로 분리

## 비범위
- 승인 절차 추가 없음
- 저장 구조 변경 없음
- CSV 추가 없음

## 완료 기준
- 일반 직원은 본인 기록만 보인다
- 관리자는 직원별 합계표를 먼저 본다
- 관리자는 `상세`를 눌러 특정 직원의 날짜별 상세내역을 본다
- 정산월 기준과 total 계산이 모두 맞는다
