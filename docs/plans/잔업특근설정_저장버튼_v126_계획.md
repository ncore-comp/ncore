# 잔업/특근 설정 저장버튼 v126 계획
# Workreport Settings Save Button v126 Plan

## 요약
## Summary
- 현재 `잔업/특근 설정(workreport)` 탭은 토글과 저장 로직은 있으나, 하단 `설정 저장` 버튼이 보이지 않습니다.
- The `잔업/특근 설정(workreport)` tab already has toggles and save logic, but the bottom `설정 저장` button is not shown.

- 원인은 `saveButton` 렌더 조건이 `settings` 탭에만 묶여 있고 `workreport` 탭이 빠져 있기 때문입니다.
- The cause is that the `saveButton` render condition only includes the `settings` tab and omits the `workreport` tab.

## 작업 범위
## Scope
- `v125`를 보존하고 `v126`으로 올립니다.
- Keep `v125` intact and bump to `v126`.

- 아래 3가지만 수정합니다.
- Only these three items will be changed.
  1. `index.html` 로딩 버전 `v126` 전환
  2. `ui_app_shell_v126_codex.js` 버전 표식 갱신
  3. `ui_permissions_v126_codex.js` 하단 저장 버튼 조건에 `workreport` 포함

## 구현 방식
## Implementation
- `permissions` 탭은 기존처럼 `권한 저장`
- `settings` 탭은 기존처럼 `설정 저장`
- `workreport` 탭도 `설정 저장`
- `mail` 탭은 기존처럼 `메일 설정 저장`

- 즉 `settings`와 `workreport`는 같은 `app.saveMasterSettings()` 버튼을 공유하도록 맞춥니다.
- In other words, `settings` and `workreport` will share the same `app.saveMasterSettings()` button.
