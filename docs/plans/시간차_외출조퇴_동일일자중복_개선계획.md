# 시간차(외출/퇴근) 동일일자 중복 개선 계획

기록 시각: 2026-03-31 07:16 KST

## 배경
- 동일 직원이 같은 날짜에 `시간차(외출)`과 `시간차(퇴근)`을 함께 사용한 사례가 확인되었다.
- 현재 구조에서는 같은 날짜의 시간차가 `실제로 가능한 조합인지`보다 `저장되었는지`만 먼저 보는 구간이 남아 있을 수 있다.

쉽게 말하면:
- 지금은 같은 날 일정표에 `외출`과 `조퇴`가 같이 적혀 있어도,
- 그 둘이 `시간상 겹치는지`, `실제로 가능한 조합인지`를 더 엄격히 보지 못한 상태다.

## 핵심 판단
### 추천 방향
- `같은 날짜 외출 + 조퇴`를 무조건 금지하지 않는다.
- 대신 아래 규칙을 추가한다.

1. 같은 날짜에 `시간차(외출)`과 `시간차(퇴근)`은 **시간이 겹치지 않을 때만** 허용
2. 같은 날짜에 `시간차(외출)`끼리도 **서로 겹치면 금지**
3. 같은 날짜에 `시간차(퇴근)`은 사실상 **1건만 허용**
4. 새 요청 시간이 기존 시간차 요청과 겹치면 저장 차단
5. 관리자 대리 등록/수정도 같은 검증을 적용

이 방향을 추천하는 이유:
- 실제 업무상 `오전 외출 + 오후 조퇴`는 가능할 수 있다.
- 하지만 `같은 시간 겹침`이나 `조퇴 2건 누적`은 데이터상 오류에 가깝다.

## 개선 목표
- 시간차 요청을 `같은 날 복수로 저장 가능`한 구조는 유지하되,
- `겹치는 시간`, `비현실적인 조합`, `중복 차감`을 막는다.

## 적용 규칙 상세
### 1. 시간차(퇴근)
- 같은 날짜에 1건만 허용
- 이미 같은 날짜 `시간차(퇴근)`이 있으면 추가 저장 차단

### 2. 시간차(외출)
- 같은 날짜 복수 허용 가능
- 단, 기존 외출 요청과 시간이 겹치면 차단

### 3. 외출 + 퇴근 동시 사용
- 같은 날짜에 둘 다 존재 가능
- 단, `외출 시간대`와 `퇴근 시간대`가 겹치면 차단

예시:
- 허용
  - 외출 `10:00~11:00`
  - 조퇴 `16:00~18:00`
- 금지
  - 외출 `15:00~17:00`
  - 조퇴 `16:00~18:00`

## 구현 순서
1. 기존 동일일자 요청 중 `시간차(외출/퇴근)` 조합 확인
2. 시간 범위 겹침 판정 함수 추가
3. 프런트 저장 전 차단 메시지 추가
4. 서버 저장 전 최종 차단 로직 추가
5. 관리자 대리 등록/수정에도 동일 규칙 적용
6. 기존 데이터 중 문제 사례가 있으면 별도 점검 목록 정리

## 사용자 안내 문구 방향
- `같은 날짜의 시간차 요청이 기존 시간과 겹칩니다. 시간을 다시 확인해 주세요.`
- `같은 날짜에는 조퇴(시간차 퇴근)를 1건만 등록할 수 있습니다.`

## 완료 기준
- 같은 날짜 외출/조퇴가 겹치면 저장되지 않는다.
- 같은 날짜 외출 + 조퇴가 서로 안 겹치면 저장 가능하다.
- 같은 날짜 조퇴 2건은 저장되지 않는다.
- 관리자 대리 등록도 같은 규칙을 따른다.
