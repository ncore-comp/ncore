# 백업 자동화 실패 보완 계획

기록 시각: 2026-03-31 07:22 KST

## 문제 요약
- 작업 스케줄러 `NCORE_Firestore_Backup_Daily`는 등록되어 있으나,
  최근 실행 결과가 `Last Result: 1`로 실패 상태였다.
- 수동으로 `backup-firestore.cmd`를 실행하면 백업은 성공했다.

쉽게 말하면:
- 백업 직원은 출근표에 올라가 있지만,
- 자동 출근했을 때는 자기 작업실이 아니라 다른 방에서 일을 시작해서 실패한 상태다.

## 원인 판단
- `backup-firestore.cmd`가 상대 경로 `node scripts\\backup-firestore.js`를 사용하고 있었다.
- 작업 스케줄러의 `Start In`이 비어 있어 실행 기준 폴더가 프로젝트 루트가 아닐 수 있다.
- 따라서 자동 실행 시 `scripts\\backup-firestore.js`를 못 찾는 문제가 발생할 가능성이 높다.

## 조치 방향
### 1. 추천
- `backup-firestore.cmd` 안에서 먼저 프로젝트 루트로 이동한 뒤 실행
- 배치파일이 어디서 실행돼도 동일하게 동작하도록 변경

### 2. 함께 보강
- `register-backup-task.ps1`도 작업 등록 시 프로젝트 루트에서 실행하도록 명령 인수를 보강

## 구현 순서
1. `backup-firestore.cmd` 절대 안전 경로 방식으로 수정
2. `register-backup-task.ps1` 보강
3. 수동 실행 검증
4. 스케줄러 수동 실행 또는 다음 실행 기준 재확인
5. 결과 MD 및 handoff 반영

## 완료 기준
- 수동 실행 성공
- 최신 백업 스냅샷 생성 확인
- 스케줄러가 같은 명령 방식으로 다시 실행 가능
- 문서에 원인과 조치가 정리됨
