# 시간차(외출) 근무시간 기반 3단 차단 계획
# Time-Off (Outing) Work-Shift-Based 3-Layer Guard Plan

## 목적
## Purpose

- `시간차(외출)`이 직원 개인의 실제 근무시간(`workQ1~workQ4`)을 기준으로만 등록되도록 정리합니다.
- Ensure `시간차(외출)` can only be registered based on each employee's actual work shift (`workQ1~workQ4`).

- 잘못된 외출 시간은
  - 화면 옵션에서 먼저 제거하고
  - 저장 직전에 다시 막고
  - 서버에서 최종 차단하는 3단 구조로 보완합니다.
- Invalid outing times will be blocked in three layers:
  - removed from UI options first,
  - blocked again before save,
  - rejected finally on the server.

## 현재 확인된 원인
## Confirmed Current Causes

1. 외출 시작시간 옵션이 공용 시간표로 생성됩니다.
1. Outing start-time options are generated from a shared fixed timetable.

- `req-start-time-out` 드롭다운이 직원별 출근시간이 아니라 고정 옵션으로 만들어집니다.
- The `req-start-time-out` dropdown is built from fixed options, not the selected employee's shift start time.

2. 대상 직원을 바꿔도 외출 시작시간 목록이 다시 걸러지지 않습니다.
2. Changing the target employee does not rebuild or filter outing start-time options.

3. 종료시간 계산이 직원 퇴근시간이 아니라 고정 기준으로 처리되고 있습니다.
3. End-time calculation is not strictly enforcing the selected employee's shift end time.

4. 서버에서도 출근 전 / 퇴근 직전 정책을 최종 검증하지 않습니다.
4. The server does not currently enforce start-before-shift / return-before-end rules as a final guard.

## 목표 규칙
## Target Rules

### 기본 규칙
### Base Rules

- 외출 시작시간은 `직원 출근시간 이상`이어야 합니다.
- The outing start time must be `at or after the employee's shift start time`.

- 외출은 `퇴근용`으로 사용할 수 없습니다.
- An outing cannot be used as a `way to leave for the day`.

- 따라서 외출 종료시간은 `직원 퇴근 1시간 전까지`여야 합니다.
- Therefore, the outing end time must be `no later than one hour before the employee's shift end time`.

- 현재 UI가 1시간 단위이므로, 마지막 허용 복귀 시각은 아래와 같습니다.
- Since the UI currently uses 1-hour increments, the last allowed return time is:
  - `lastAllowedReturnHour = shift.endHour - 1`

- 예: `09:00 ~ 18:00` 근무 직원은
- Example: for an employee working `09:00 ~ 18:00`
  - `07:00 시작` 불가
  - `07:00 start` is invalid
  - `17:00 ~ 18:00` 불가
  - `17:00 ~ 18:00` is invalid
  - `16:00 ~ 17:00` 가능
  - `16:00 ~ 17:00` is valid

### 점심시간 처리
### Lunch-Break Handling

- 현재 로직처럼 `12:00`은 외출 시간 차감 계산에서 점심시간으로 건너뜁니다.
- Keep the current lunch behavior where `12:00` is skipped in duration calculation.

- 단, 점심시간 처리와 별개로 허용 범위는 반드시 `출근시간 ~ (퇴근 1시간 전)` 안에 있어야 합니다.
- Regardless of lunch logic, the allowed range must still remain within `shift start ~ one hour before shift end`.
