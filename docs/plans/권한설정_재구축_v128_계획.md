## 4. 재구축 원칙
## 4. Rebuild Principles

1. 분할 구조는 유지한다.
1. Keep the split architecture.

2. `권한/설정`만 독립적으로 다시 세운다.
2. Rebuild only the `master permissions` module independently.

3. 정상 한글 문자열이 남아 있는 모놀리식 구간을 기준선으로 삼는다.
3. Use a monolithic-era source that still has clean Korean strings as the textual baseline.

4. 최신 기능은 한 번에 다 합치지 않고, 작은 묶음으로 다시 얹는다.
4. Reapply recent behavior in small bundles, not all at once.

5. 각 탭이 살아난 뒤 다음 탭으로 넘어간다.
5. Move to the next tab only after the current one is stable.

## 8. 위험 요소
## 8. Risks

1. 기준선 추출 시 `shared`로 이미 옮긴 함수와 중복될 수 있다.
1. Re-extraction may duplicate functions already moved into shared modules.

2. 운영실은 최근 로그 절감 변경이 들어가 있어, 옛 기준선을 그대로 쓰면 다시 읽기 문제가 생길 수 있다.
2. Admin ops recently received log-loading optimizations, so naïvely restoring an old block may reintroduce read-cost issues.

3. 게시판 권한은 `v127`에서 새로 들어간 기능이라, 재적용 누락 시 다시 마스터/직원 차이가 재발할 수 있다.
3. Board permissions were added in `v127`, so missing that reapply may reintroduce master/employee mismatches.
