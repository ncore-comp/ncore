# 4월 9일 시연 계획

## 1. 목적

- 4월 9일 시연의 목표는 `이 시스템이 실제 운영 가능한 상태`임을 짧고 명확하게 보여주는 것이다.
- 즉 `예쁜 화면 소개`보다 `실제 업무가 어떻게 흘러가는지`를 보여주는 데 집중한다.

쉽게 비유하면:
- 이번 시연은 `집 소개`가 아니라 `실제로 문 열고, 전등 켜고, 수도 틀고, 잠금장치 확인하는 입주 점검`이다.

## 2. 시연의 핵심 메시지

시연에서 꼭 전달해야 할 메시지는 아래 4개다.

1. 직원이 직접 신청하고, 관리자/대표가 바로 확인할 수 있다.
2. 관리자 운영실에서 누락 데이터 보정, 수동 휴일 설정, 로그 다운로드까지 가능하다.
3. Firebase 기준으로 운영 안정화, 백업/복구, 동시성 테스트까지 끝난 상태다.
4. 지금은 `실사용 가능한 운영판`이며, 남은 것은 장기 개선 과제 수준이다.

## 4. 시연 시나리오

### 4-1. 오프닝

설명 포인트:
- 이 시스템은 기존 시트/수기 운영을 Firebase 기반 웹 운영으로 옮긴 결과물이다.
- 현재는 실제 운영 가능한 수준으로 안정화되어 있다.

짧은 멘트 예시:
- `이번 시연은 화면 소개보다, 실제 근태/연차 업무가 어떻게 처리되는지 보여드리는 데 집중하겠습니다.`

### 4-2. 일반 직원 신청 흐름

보여줄 것:
1. 로그인
2. 연차 신청
3. 반차/시간차 신청
4. 지난 날짜 신청 시 상세사유가 추가로 필요한 흐름

강조 포인트:
- 일반 신청은 직관적이다.
- 지난 날짜/지난 시간 신청은 상세사유를 받도록 통제돼 있다.
- 시간차는 시간 범위가 명시적으로 보인다.

추천 시연 예:
- 연차 1건
- 시간차(외출) 1건
- 지난 날짜 신청 시 상세사유 입력 1건

### 4-3. 승인/확인 흐름

보여줄 것:
1. 결재 대기 확인
2. 승인 처리
3. 새로고침 후 상태 유지 확인

강조 포인트:
- 예전처럼 화면만 바뀌고 DB가 안 바뀌는 문제가 아니라, 실제 DB 상태가 유지된다.
- 승인 후 대표/관리자 화면에도 반영된다.

### 4-4. 대표용 전체상황판

보여줄 것:
1. 전체상황판 진입
2. 날짜별 요약 칩
3. `반차`, `시간차(퇴근)`, `시간차(외출)` 분리 표시
4. 전체 자리배치도
5. 오늘 날짜일 때 현재 시각 기준으로 실제 부재 인원만 보이는 점

강조 포인트:
- 연차 / 반차 / 시간차가 서로 섞이지 않는다.
- 자리배치도는 단순 일정표가 아니라, 오늘은 현재 시각 기준으로 실제 부재를 반영한다.

### 4-5. 관리자 운영실

보여줄 것:
1. 최근 접속 로그
2. 접속 로그 CSV 다운로드
3. 대리 등록 / 수정 / 삭제
4. 수동 휴일 설정
5. 공휴일 동기화

강조 포인트:
- 운영실은 `관리자가 DB를 직접 만지지 않고도` 주요 운영작업을 할 수 있도록 만든 공간이다.
- 대리 등록은 `결재 대기`가 아니라 `관리자 직행 등록`이다.
- 공휴일은 자동 + 수동 병행 구조다.

추천 시연 예:
- 특정 직원 일정 1건 대리 등록
- 기존 일정 1건 수정
- 수동 휴일 1건 추가 후 제거
- 로그 CSV 다운로드 버튼 확인

## 7. 시연용 권장 동선

가장 자연스러운 시연 순서는 아래다.

1. 직원 로그인
2. 신청 등록
3. 승인권자 승인
4. 대표가 전체상황판 확인
5. 관리자 운영실에서 보정 기능 시연

이 순서가 좋은 이유:
- 신청 → 승인 → 조회 → 운영 보정
- 즉 실제 회사 업무 흐름과 같다.

## 8. 질문이 나올 가능성이 높은 항목과 답변 방향

### Q. 운영해도 되나?

답변 방향:
- 현재는 운영 가능한 상태다.
- 시연 전수 테스트, 백업/복구 리허설, 운영실 기능, 전체상황판 정리까지 완료되어 있다.

### Q. 공휴일은 어떻게 관리하나?

답변 방향:
- 대한민국 기본 공휴일은 동기화
- 회사 휴일은 수동 추가
- 두 구조를 함께 사용한다.

### Q. 잘못 입력된 데이터는 어떻게 고치나?

답변 방향:
- 운영실에서 관리자 대리 등록/수정/삭제로 직접 보정 가능
- DB를 직접 여는 방식이 아니라 운영실에서 처리 가능

### Q. 로그는 남나?

답변 방향:
- 접속 로그가 남고
- CSV 다운로드도 가능하다.

쉽게 말하면:
- `기능 자랑`보다 `실제 업무가 잘 굴러간다`는 인상을 주는 쪽이 더 중요하다.

### 마무리

## 11. 플랜 B

시연 도중 한 기능이 꼬이면 바로 우회할 수 있어야 한다.

우회 원칙:
1. 로그인 꼬이면 미리 로그인된 브라우저 탭 사용
2. 신청 꼬이면 준비된 기존 승인 건으로 설명 전환
3. 운영실에서 꼬이면 CSV 다운로드 / 휴일 설정처럼 짧고 안정적인 기능으로 전환
4. 전체상황판은 월간 달력 → 자리배치도 순서로 단순 시연

## 12. 성공 기준

아래가 되면 시연은 성공이다.

1. 직원 신청 흐름이 막힘 없이 보인다.
2. 승인 후 상태 유지가 확인된다.
3. 대표 화면에서 전체상황판 가치가 전달된다.
4. 관리자 운영실이 `실제 운영 도구`처럼 보인다.
5. 참석자가 `이제 실사용 가능한 시스템`이라고 받아들인다.
