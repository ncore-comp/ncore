# 권한/설정 팀 필터 계획

## 1. 목적

- `권한/설정 > 권한 탭`에서 사용자 목록을 `전부 / 매뉴얼팀 / 파츠북팀` 기준으로 나눠서 볼 수 있게 한다.
- 현재는 모든 직원이 한 표에 섞여 있어서, 팀별 권한 점검이나 수정이 길어지고 헷갈릴 수 있다.

쉽게 비유하면:
- 지금은 한 권짜리 큰 장부를 통째로 펼쳐 보는 상태다.
- 여기에 `전부 / 매뉴얼팀 / 파츠북팀` 탭을 붙여서 필요한 장만 넘겨보는 구조로 바꾸는 작업이다.

## 2. 현재 상태

현재 권한 탭에는 이미 아래 토글이 있다.

- `달력정보`
- `승인권한`
- `팀원 연차현황`

이 토글은 `열 그룹`을 바꾸는 역할이다.

즉:
- 지금 있는 토글 = 어떤 권한 종류를 볼지 정함
- 이번에 추가할 토글 = 어떤 팀 사람들을 볼지 정함

## 3. 목표 화면

권한표 상단 오른쪽에 아래 3개 버튼을 둔다.

1. `전부`
2. `매뉴얼팀`
3. `파츠북팀`

버튼을 누르면:
- 표의 `행(row)`만 필터링된다.
- 기존 `달력정보 / 승인권한 / 팀원 연차현황` 열 토글은 그대로 유지된다.

## 4. 구현 방향

### 4-1. 상태값 추가

새 상태값 예시:

- `permissionDeptFilter = 'all'`

허용값:

- `all`
- `manual`
- `parts`

기본값:

- `all`

### 4-2. 버튼 UI 추가

위치:
- 현재 열 그룹 토글(`달력정보 / 승인권한 / 팀원 연차현황`) 오른쪽 영역

형태:
- `전부`
- `매뉴얼팀`
- `파츠북팀`

선택된 버튼은 진하게, 비선택 버튼은 옅게 표시

### 4-3. 렌더링 필터

권한표에서 `users` 배열을 바로 행으로 그리지 않고,
먼저 부서 기준으로 거른 뒤 행을 만든다.

예시 규칙:

- `all` → 전체 사용자
- `manual` → `user.dept === '매뉴얼팀'`
- `parts` → `user.dept === '파츠북팀'`

대표/마스터는 사용자 요청 기준으로 별도 탭이 없으므로:
- `전부`에서만 보이게 두는 방향이 자연스럽다.

## 5. 저장 원칙

중요 원칙:
- 화면에 보이는 행만 편하게 볼 뿐,
- 저장은 현재처럼 전체 권한 데이터를 기준으로 동작한다.

즉:
- 필터는 `보기 편의`
- 저장 로직은 `기존 유지`

이유:
- 팀 필터 때문에 다른 팀 데이터가 저장에서 누락되면 안 된다.

## 6. 사용자 동작 기준

### 예시 1

- `매뉴얼팀` 버튼 클릭
- 표에는 매뉴얼팀 직원만 보임
- 체크 수정 후 저장
- 파츠북팀 데이터는 화면에서 숨겨져 있을 뿐, 저장에서 사라지지 않음

### 예시 2

- `파츠북팀` 버튼 클릭
- 파츠북팀 직원만 보임
- 권한 수정 후 저장

### 예시 3

- `전부` 클릭
- 전체 사용자 다시 표시

## 7. 예외/주의

1. 팀 필터는 `권한 탭`에만 적용
   - 설정 테이블 / 메일 설정 / 운영실에는 영향 없음

2. 저장 시 필터 때문에 다른 팀 데이터가 손상되면 안 됨
   - 기존 저장 로직을 건드리지 않는 쪽으로 구현

3. 대표/마스터 계정은
   - 부서 기준 필터에서 애매할 수 있으므로
   - 기본적으로 `전부`에서 확인하는 것이 자연스러움

## 8. 검증 계획

1. `전부` 버튼 클릭 시 전체 직원 표시 확인
2. `매뉴얼팀` 버튼 클릭 시 매뉴얼팀만 표시 확인
3. `파츠북팀` 버튼 클릭 시 파츠북팀만 표시 확인
4. 필터 전환 후 기존 체크값 유지 확인
5. 필터 상태에서 권한 저장 후 저장 정상 확인

## 9. 완료 기준

아래가 되면 완료로 본다.

1. 권한표 상단 오른쪽에 `전부 / 매뉴얼팀 / 파츠북팀` 버튼이 보인다.
2. 각 버튼 클릭 시 팀별 행 필터가 정확히 동작한다.
3. 기존 열 그룹 토글과 충돌하지 않는다.
4. 저장 로직이 깨지지 않는다.

## 10. 추천 판단

지금 단계에서는 `검색`까지 붙이는 확장안보다,
우선 `전부 / 매뉴얼팀 / 파츠북팀` 3버튼만 넣는 것이 가장 적절하다.

즉:
- 구조는 단순하게
- 효과는 바로 체감되게
가는 것이 맞다.
