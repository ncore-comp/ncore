# 파일 버전 네이밍 계획
# File Version Naming Plan

## 목표
## Goal

- 현재 운영 기준 JS 파일명을 `v001` 세트로 다시 시작한다.
- Restart the current production JS file naming as a clean `v001` set.

- 이후 수정은 `v002`, `v003`처럼 같은 규칙으로 올려가며 관리한다.
- Manage future changes by incrementing with the same rule, such as `v002`, `v003`, and so on.

- 과거 버전 파일은 루트 `js`에 계속 쌓지 않고 `js\legacy_versions`로 분리한다.
- Stop accumulating old version files in the root `js` folder and move them into `js\legacy_versions`.

## 기본 원칙
## Core Rules

1. 현재 운영본은 모두 `v001`로 통일한다.
1. Standardize the current production set as `v001`.

2. 이후 운영 수정 시, 실제 배포 대상으로 바뀌는 파일만 같은 새 버전 번호로 올린다.
2. For later production changes, only the files that actually change for deployment should move together to the next version number.

3. 과거 버전은 삭제하지 않고 `legacy_versions`로 먼저 보관한다.
3. Do not delete old versions immediately; archive them in `legacy_versions` first.

4. `index.html`는 항상 현재 운영본 세트만 읽도록 유지한다.
4. Keep `index.html` pointing only to the current production set.

## 폴더 구조
## Folder Structure

- 현재 운영본:
- Current production:
  - `js\*_v001_codex.js`

- 과거 버전 보관:
- Old version archive:
  - `js\legacy_versions\<family>\원래파일명`

예:
Example:

- `js\legacy_versions\ui_dashboard\ui_dashboard_v143_codex.js`
- `js\legacy_versions\ui_logic_new_design\ui_logic_new_design_v150_codex.js`

## 정리 순서
## Cleanup Order

1. 현재 운영 파일을 새 `v001` 이름으로 복사한다.
1. Copy the current production files under the new `v001` names.

2. `index.html`를 새 `v001` 세트로 교체한다.
2. Update `index.html` to use the new `v001` set.

3. 기존 `_vNNN_codex.js` 파일은 현재본을 제외하고 `js\legacy_versions`로 이동한다.
3. Move existing `_vNNN_codex.js` files, except for the new current set, into `js\legacy_versions`.

4. 문서 링크와 기록 문서를 새 구조에 맞게 갱신한다.
4. Update document links and record files to match the new structure.

5. hosting 재배포 후 실제 운영 HTML이 `v001`을 읽는지 확인한다.
5. Redeploy hosting and confirm that the live HTML loads the `v001` files.

## 이후 작업 규칙
## Rule For Future Work

- 다음 수정부터는 새 운영본이 되는 파일만 `v002`로 올린다.
- Starting with the next change, only the files that become the new production set should be bumped to `v002`.

- 같은 릴리즈 안에서 여러 파일을 바꿨다면 같은 버전 번호를 맞춘다.
- If multiple files are changed in the same release, keep them on the same version number.

- 버전 올림 후 직전 운영본은 `legacy_versions`로 이동한다.
- After a version bump, move the previous production files into `legacy_versions`.

## 완료 기준
## Definition of Done

- `index.html`가 `v001` 세트를 읽는다.
- `index.html` loads the `v001` set.

- 이전 버전 파일은 `js\legacy_versions`로 분리된다.
- Previous version files are separated into `js\legacy_versions`.

- 현재 운영 기준 파일 목록을 문서에서 바로 확인할 수 있다.
- The current production file list can be checked directly from documentation.
