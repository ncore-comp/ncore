# 출산휴가 법정 규칙 계획
# Maternity Leave Statutory Rules Plan

## 1. 목적
## 1. Purpose

- 출산휴가를 일반 특별휴가와 분리하고, 법정 규칙에 맞는 별도 워크플로우로 설계한다.
- Separate maternity leave from ordinary special leave and design it as an independent statutory leave workflow.

- 임의 설정값이 아니라 법 기준을 우선하는 구조로 고정한다.
- Use a law-first structure rather than admin-defined discretionary rules.

## 2. 범위
## 2. Scope

- 본 문서는 `출산전후휴가`를 대상으로 한다.
- This document covers `maternity leave before and after childbirth`.

- 본 문서는 일반 경조사 특별휴가, 병가, 배우자 출산휴가를 직접 다루지 않는다.
- This document does not directly cover ceremonial special leave, sick leave, or spouse childbirth leave.

## 4. 핵심 법정 규칙
## 4. Core Statutory Rules

- 일반 출산전후휴가 총 기간은 `90일`로 본다.
- The standard maternity leave duration is `90 days`.

- 다태아 출산은 총 `120일`로 본다.
- Multiple births use a total of `120 days`.

- 미숙아 출산은 총 `100일`로 본다.
- Premature infant childbirth uses a total of `100 days`.

- 산후 최소 확보 기간은 일반 `45일`, 다태아 `60일`로 본다.
- The minimum guaranteed postnatal period is `45 days` for standard cases and `60 days` for multiple births.

- 출산일은 산후 기간 계산에 포함되는 일정 처리 규칙을 별도 구현 단계에서 확인하되, UI/검증은 `산후 최소 확보일 위반 금지` 기준으로 설계한다.
- The exact counting treatment of the childbirth date should be confirmed during implementation, but the UI and validations must enforce the `minimum postnatal days cannot be violated` rule.

- 출산 전 분할 사용은 법정 허용 사유가 있을 때만 허용한다.
- Pre-birth split usage is allowed only when statutory conditions are met.

- 법정 허용 사유 예시
- Examples of statutory split-usage eligibility
  - 유산·사산 경험
  - prior miscarriage or stillbirth
  - 만 40세 이상
  - age 40 or older
  - 유산·사산 위험 진단서 제출
  - medical certificate showing miscarriage/stillbirth risk

## 5. 시스템 정책
## 5. System Policy

- 출산휴가는 `특별휴가 종류 설정`에서 임의로 만들고 지우는 항목으로 두지 않는다.
- Maternity leave must not remain as a freely created/deleted item inside the ordinary special-leave type settings.

- 출산휴가는 별도 타입군으로 분리한다.
- Maternity leave must be split into its own leave family.

- 신청 단위는 `기간형`만 허용한다.
- Only `date-range` requests are allowed.

- `반차`, `시간차(외출)`, `시간차(퇴근)`로는 신청할 수 없게 한다.
- Disallow `half-day`, `time-off(out)`, and `time-off(leave early)` request modes.

- 일반 연차처럼 `시간(hour)` 차감형으로 계산하지 않는다.
- Do not calculate maternity leave like ordinary annual leave with hour-based deduction.

- 출산휴가는 `일(day)` 단위 누적 사용으로 관리한다.
- Manage maternity leave by cumulative `day` usage.

## 6. 권장 타입 설계
## 6. Recommended Type Design

- 권장 타입
- Recommended types
  - `maternity_standard_90`
  - `maternity_multiple_120`
  - `maternity_premature_100`

- 각 타입은 관리자 임의 입력이 아니라 시스템 정의값으로 둔다.
- Each type should be a system-defined option, not a free-form admin-created type.

## 7. 신청 워크플로우
## 7. Request Workflow

- 신청자는 `출산휴가`를 선택한다.
- The requester selects `maternity leave`.

- 시스템은 아래 필수 입력을 받는다.
- The system collects the following required inputs.
  - 출산 유형
  - childbirth type
  - 출산 예정일 또는 출산일
  - expected childbirth date or actual childbirth date
  - 분할 사용 사유 여부
  - whether split usage is legally justified
  - 관련 증빙 여부
  - whether supporting evidence exists

- 시스템은 총 허용일을 자동 계산한다.
- The system auto-calculates the total entitled days.

- 시스템은 신청 기간이 산후 최소 확보일을 침해하면 저장을 차단한다.
- The system blocks saving when the requested range violates the minimum protected postnatal days.

- 시스템은 출산 전 분할 사용이 법정 사유 없이 시도되면 저장을 차단한다.
- The system blocks pre-birth split usage when no statutory justification is supplied.

## 8. 승인 워크플로우
## 8. Approval Workflow

- 승인 흐름은 일반 연차와 UI를 공유할 수 있지만, 검증 규칙은 별도로 둔다.
- The approval flow may share UI with annual leave, but validation rules must be separate.

- 승인자는 법정 필수 조건 위반 요청을 승인할 수 없게 한다.
- Prevent approvers from approving requests that violate mandatory statutory rules.

- 승인/반려/취소 상태값은 일반 연차 체계를 재사용할 수 있다.
- Approval/rejection/cancel status values may reuse the annual-leave state model.

## 9. 데이터 모델 방향
## 9. Data Model Direction

- 별도 컬렉션 또는 별도 request subtype을 둔다.
- Use either a dedicated collection or a dedicated request subtype.

- 최소 필요 필드
- Minimum required fields
  - `maternityLeaveType`
  - `entitledDays`
  - `usedDays`
  - `childbirthDate`
  - `isSplitUsage`
  - `splitUsageReason`
  - `evidenceRequired`
  - `postnatalProtectedDays`

- 일반 특별휴가의 `grantId` 구조를 그대로 재사용하지 않는다.
- Do not directly reuse the ordinary special-leave `grantId` model.

## 10. UI 권장안
## 10. UI Recommendation

- `연차 신청` 모달에 출산휴가 전용 분기 UI를 둔다.
- Add a maternity-leave-specific branch UI inside the leave request modal.

- 출산휴가를 선택하면 일반 특별휴가 드롭다운을 감추고, 전용 입력을 보여준다.
- When maternity leave is selected, hide the ordinary special-leave dropdown and show dedicated inputs.

- 입력 중 실시간으로 아래를 안내한다.
- Show the following in real time while the user is entering data.
  - 총 허용일
  - total entitled days
  - 현재 신청일수
  - currently requested days
  - 산후 최소 확보 남은 일수
  - remaining protected postnatal days
  - 저장 가능 / 불가 사유
  - save allowed / blocked reason

## 11. 테스트 기준
## 11. Test Criteria

- 일반 출산 `90일` 케이스
- Standard `90-day` maternity leave case

- 다태아 `120일` 케이스
- Multiple-birth `120-day` case

- 미숙아 `100일` 케이스
- Premature-infant `100-day` case

- 산후 `45일` 미만으로 남게 하는 신청 차단
- Block requests that leave fewer than `45 postnatal days`

- 다태아 산후 `60일` 미만 신청 차단
- Block multiple-birth requests that leave fewer than `60 postnatal days`

- 법정 사유 없이 출산 전 분할 사용 차단
- Block pre-birth split usage without a legal justification

- 반차/시간차 신청 차단
- Block half-day/time-off request modes
