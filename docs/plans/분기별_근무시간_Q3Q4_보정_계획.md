## 1. 목적

현재 일부 직원 데이터는 다음과 같은 상태다.

- `workQ1` 입력됨
- `workQ2` 입력됨
- `workQ3` 비어 있음
- `workQ4` 비어 있음

현재 시스템은 분기 값이 비어 있으면 다른 분기 값으로 fallback 하므로,
운영상 혼선을 줄이기 위해 비어 있는 `Q3/Q4`를 `Q2` 기준으로 미리 채워둔다.

쉽게 비유하면:

- 1분기, 2분기 표는 붙어 있는데
- 3분기, 4분기 표 칸이 비어 있어서
- 사람들이 임시 규칙으로 해석하게 되는 상태다
- 이번 작업은 그 빈 칸에 `2분기 표를 그대로 복사해서 붙이는 작업`이다

## 2. 적용 원칙

### 2-1. 수정 대상

다음 조건을 모두 만족하는 직원만 수정한다.

1. `workQ2` 값이 존재함
2. `workQ3`가 비어 있음
3. 또는 `workQ4`가 비어 있음

### 2-2. 수정 규칙

- `workQ3`가 비어 있으면 `workQ2` 값으로 채움
- `workQ4`가 비어 있으면 `workQ2` 값으로 채움

### 2-3. 수정하지 않는 경우

- `workQ2` 자체가 비어 있는 직원
- 이미 `workQ3` 또는 `workQ4`에 값이 있는 직원

## 3. 실행 순서

1. 현재 Firestore 백업 1회 실행
2. 대상 직원 목록 조회
3. `Q2 -> Q3/Q4` 보정 스크립트 실행
4. 변경 건수 확인
5. 샘플 직원 데이터 재조회
6. 결과 문서 작성

## 4. 예상 결과

- 빈 `Q3/Q4`가 더 이상 fallback에 의존하지 않음
- 2026년 하반기와 2027년 이후도 `Q2 기준` 근무시간이 명시적으로 들어감
- 운영실 수정, 시간차(퇴근), 반차 계산이 더 읽기 쉬운 데이터 상태가 됨

## 5. 완료 기준

아래를 만족하면 완료로 본다.

1. 백업이 생성됨
2. 대상 직원만 보정됨
3. `Q3/Q4` 비어 있던 직원이 `Q2` 값으로 채워짐
4. 결과 문서에 변경 건수와 샘플 검증이 기록됨
