# PWA 우선도입 계획

## 1. 방향

- 초보자인 현재 상황에서 가장 쉬운 모바일앱 첫 단계는 `PWA`다.
- 즉 지금 웹을 바로 버리지 않고, `앱처럼 설치해서 쓰는 웹앱`으로 먼저 가는 전략이다.

쉽게 비유하면:
- 아직 정식 앱 공장을 돌리는 게 아니라
- 지금 웹 매장에 `홈 화면 바로가기`, `앱 아이콘`, `앱 이름`을 붙여서 휴대폰에서 앱처럼 쓰게 만드는 단계다.

## 2. 왜 PWA부터 가는가

현재 조건:
- Mac 없음
- iPhone App Store 등록은 지금 바로 어렵다
- 초보자 기준으로 가장 빨리 결과를 보려면 웹 자산 재사용이 중요하다

PWA 장점:
1. 현재 웹/Firebase 구조를 거의 그대로 사용
2. 설치형처럼 체험 가능
3. iPhone / Android 둘 다 `앱처럼` 써볼 수 있음
4. 실패 부담이 작음
5. 이후 안드로이드 앱/아이폰 앱으로 가기 전 모바일 문제를 먼저 볼 수 있음

## 3. PWA로 얻는 것

PWA를 붙이면 사용자 입장에서는:

1. 홈 화면에 아이콘 추가 가능
2. 브라우저 탭 느낌이 아니라 앱 창처럼 실행 가능
3. 주소창이 줄어들거나 숨겨진 앱 느낌으로 사용 가능

즉:
- 완전한 앱스토어 앱은 아니지만
- `앱처럼 쓰는 경험`을 먼저 만들 수 있다

## 4. 이번 단계 목표

이번 PWA 1차 목표는 아래다.

1. 홈 화면에 설치 가능
2. 앱 이름/아이콘 지정
3. 앱처럼 전체 화면에 가깝게 실행
4. 기본 페이지 재방문 속도 개선 가능성 확보

## 5. 구현 항목

### 1단계. 앱 메타데이터 준비

필요한 것:
- 앱 이름
- 짧은 이름
- 아이콘
- 테마 색
- 배경색

예:
- 앱 이름: `NCORE 연차관리`
- 짧은 이름: `NCORE`

### 2단계. `manifest.json` 추가

역할:
- 앱 이름
- 아이콘
- 시작 URL
- 색상
- 표시 방식

즉:
- 앱 명함을 만드는 단계

### 3단계. `service worker` 추가

역할:
- 정적 자산 캐시
- 재방문 속도 향상
- 기본 오프라인 대응 기반

주의:
- 처음부터 복잡한 오프라인 기능까지 욕심내지 않는다
- 1차는 `앱 설치 + 기본 캐시` 정도만

### 4단계. `index.html` 연결

추가할 것:
- manifest 연결
- theme-color meta
- iPhone용 메타 태그

### 5단계. 설치/실행 점검

확인할 것:
- Android Chrome에서 설치 가능 여부
- iPhone Safari에서 홈 화면 추가 가능 여부
- 앱 아이콘 표시
- 독립 실행 화면 느낌 확인

## 6. iPhone/Android 차이

### Android

- PWA 지원이 더 좋음
- 설치 버튼/설치 경험이 비교적 자연스러움

### iPhone

- 보통 Safari의 `홈 화면에 추가` 방식
- App Store 앱은 아님
- 그래도 앱처럼 실행 가능

즉:
- Android 쪽이 먼저 체감이 좋고
- iPhone은 설치 UX가 조금 다르지만 충분히 테스트 가능

## 7. 이번 단계에서 하지 않을 것

1. App Store 등록
2. Play Store 등록
3. 푸시 알림
4. 복잡한 완전 오프라인 모드
5. 네이티브 기능 연동

즉 이번 단계는:
- `앱 흉내를 잘 내는 웹`
까지를 목표로 한다.

## 8. 체크리스트

### 개발 체크

1. `manifest.json` 생성
2. 아이콘 파일 준비
3. `service-worker.js` 생성
4. `index.html` 연결
5. Firebase Hosting 배포

### 검증 체크

1. Android Chrome 설치 가능
2. iPhone Safari 홈 화면 추가 가능
3. 앱 아이콘 정상
4. 앱 이름 정상
5. 실행 후 기본 화면 정상
6. 로그인 정상

## 9. 완료 기준

아래가 되면 PWA 1차 도입 완료로 본다.

1. 모바일에서 `설치/홈화면 추가` 가능
2. 설치 후 아이콘 실행 가능
3. 로그인 및 핵심 화면 진입 가능
4. 현재 웹 기능과 충돌 없음

## 10. 다음 단계

PWA 1차가 끝나면 다음 두 갈래로 간다.

1. Android 앱 포장(Capacitor)
2. iOS 최종 준비를 위한 모바일 UX 보정

즉:
- PWA는 최종 목적지가 아니라
- `모바일앱으로 가는 가장 쉬운 첫 번째 계단`이다.

## 11. 결론

초보자인 지금 가장 쉬운 방향은
**`PWA를 먼저 붙여서 휴대폰에서 앱처럼 설치/실행되는 경험을 만들고, 그 다음 안드로이드 앱, 나중에 iPhone App Store로 가는 것`**이다.
