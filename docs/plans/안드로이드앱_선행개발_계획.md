# 안드로이드앱 선행개발 계획

## 1. 전제

- 최종 목표는 `아이폰 App Store 등록`이다.
- 하지만 현재는 Mac이 없어서 iOS 빌드/제출을 바로 진행할 수 없다.
- 따라서 지금 할 수 있는 가장 현실적인 경로는 **안드로이드 앱부터 먼저 만들고, 모바일 이슈를 선행 정리한 뒤 나중에 iOS로 확장하는 것**이다.

쉽게 비유하면:
- 최종 목적지는 `아이폰 출고`지만,
- 지금은 아이폰 공장이 없으니
- 같은 차체로 `안드로이드 조립 라인`에서 먼저 시운전하는 전략이다.

## 2. 목표

1. 현재 웹 프로젝트를 기반으로 안드로이드 앱 형태를 만든다.
2. 모바일 환경에서 생기는 UI/UX 문제를 먼저 잡는다.
3. 나중에 Mac이 생겼을 때 같은 구조로 iOS 앱으로 확장할 수 있게 준비한다.

## 3. 권장 방식

추천 방식:
- `Capacitor 기반 웹뷰 앱`

이유:
- 현재 웹 자산 재사용률이 가장 높다.
- Firebase 구조를 그대로 유지하기 쉽다.
- 안드로이드와 iOS를 같은 방향으로 묶어갈 수 있다.

즉:
- 새 집을 짓는 게 아니라
- 지금 집에 모바일 출입문을 다는 방식이다.

## 4. 단계별 계획

### 1단계. 앱 전환 준비

할 일:
- 현재 웹 프로젝트 구조를 앱 전환 기준으로 점검
- 모바일에서 민감한 기능 목록 정리
  - 로그인
  - 세션 유지
  - 연차 신청
  - 승인/반려
  - 운영실
  - 전체상황판
  - 메일 호출
  - CSV 다운로드

산출물:
- 모바일 우선 점검 목록

### 2단계. Capacitor 프로젝트 생성

할 일:
- Capacitor 초기화
- 현재 웹 자산 연결
- Android 플랫폼 추가
- 앱 이름 / 앱 아이콘 / 기본 설정 연결

산출물:
- 안드로이드 앱 프로젝트 뼈대

### 3단계. 안드로이드 1차 실행

할 일:
- 에뮬레이터 또는 실기기에서 실행
- 앱 구동, 로그인, 기본 화면 진입 확인

산출물:
- 실행 가능한 안드로이드 앱 초안

### 4단계. 모바일 UX 점검

우선 확인할 것:
1. 로그인 유지
2. 모달 크기
3. 버튼 터치 영역
4. 달력 스크롤/탭 조작
5. 전체상황판 가독성
6. 운영실 사용성
7. CSV 다운로드 동작
8. 메일 초안 호출 동작

산출물:
- 모바일 이슈 목록

### 5단계. 보정 작업

할 일:
- 모바일에서 불편한 부분 수정
- 앱 환경에서 막히는 기능 보완

산출물:
- 안드로이드 실사용 보정판

### 6단계. Android 내부 테스트

권장 테스트:
- 일반직원
- 승인권자
- 대표/마스터

시나리오:
- 로그인
- 신청
- 승인
- 운영실
- 전체상황판

산출물:
- 안드로이드 앱 내부 테스트 결과

### 7단계. iOS 준비 패키지 정리

Mac이 생기거나 외부 도움을 받을 때를 대비해 정리할 것:
- 앱 이름
- 앱 아이콘
- 스플래시
- 번들 ID 후보
- 앱 설명문 초안
- 테스트 시나리오
- 앱 권한 설명 문구

산출물:
- iOS 전환 준비 문서

## 5. 예상 준비물

### 지금 바로 필요한 것

1. 현재 프로젝트 소스
2. Android Studio
3. Capacitor 관련 npm 패키지
4. 안드로이드 테스트 단말 또는 에뮬레이터

### 나중에 iOS로 갈 때 필요한 것

1. Mac
2. Xcode
3. Apple Developer Program 계정

## 6. 예상 리스크

1. 웹에서 자연스럽던 기능이 앱에서 다르게 보일 수 있음
   - CSV 다운로드
   - 메일 호출
   - 외부 앱 연동

2. 모바일 화면에서 모달/달력 조작이 불편할 수 있음

3. 앱으로 감싼 뒤 세션/브라우저 저장 방식이 다르게 느껴질 수 있음

## 7. 우선순위

가장 먼저 해야 할 것은:

1. Capacitor 도입 가능 여부 정리
2. 안드로이드 프로젝트 뼈대 생성
3. 앱 구동 확인

즉:
- 처음부터 앱 완성형을 노리지 말고
- `안드로이드에서 일단 뜨는 것`을 1차 목표로 잡는 게 맞다.

## 8. 완료 기준

아래가 되면 1차 목표 달성으로 본다.

1. 안드로이드에서 앱이 구동된다.
2. 로그인 후 주요 화면 진입이 된다.
3. 핵심 기능 흐름이 모바일에서도 끊기지 않는다.
4. 나중에 iOS로 확장 가능한 구조가 확보된다.

## 9. 결론

최종 목표는 아이폰 앱이지만,
현재 Mac이 없는 상황에서는 **안드로이드 앱을 먼저 만들고 모바일 문제를 선행 해결하는 전략이 가장 현실적**이다.

즉:
- 지금은 안드로이드 선행 개발
- 나중에 Mac 확보 후 iOS 확장

으로 가는 것이 맞다.
