# 달력정보 UI 가독성 개선 계획

## 작업 일시
- 2026-04-01

## 작업 배경
- `달력정보` 권한 항목이 아래 6개로 늘어났습니다.
  - 자기연차
  - 매뉴얼팀
  - 파츠북팀
  - 모두
  - 반려
  - 잔업/특근
- 현재는 이 6개를 가로 1줄로 배치하고 있어, 화면 폭에 따라 한눈에 보이지 않고 가로 스크롤이 필요해졌습니다.
- 권한표의 장점이었던 `한 번에 보고 바로 체크`가 약해진 상태입니다.

## 목표
- `달력정보` 체크박스를 가로 스크롤 없이 한눈에 볼 수 있도록 재배치
- 기존 권한 의미와 저장 구조는 바꾸지 않고, **UI 배치만 개선**

## 추천 방향
### 2안
- `달력정보` 6개 체크박스를 `2줄 x 3칸` 구조로 변경

### 배치안
- 1줄:
  - 자기연차
  - 매뉴얼팀
  - 파츠북팀
- 2줄:
  - 모두
  - 반려
  - 잔업/특근

## 이유
- 가로로 길게 늘어나는 구조를 멈출 수 있음
- 스크롤 없이 한눈에 체크 상태 확인 가능
- 체크박스 클릭 영역을 충분히 유지할 수 있음
- 나중에 항목이 조금 더 늘어도 대응하기 쉬움

## 구현 방향
1. `달력정보` 헤더를 6칸 가로형에서 `2줄 3칸` 레이아웃으로 변경
2. 각 직원 행의 체크박스 묶음도 `2줄 3칸` 레이아웃으로 변경
3. 왼쪽 기본 정보 열(`이름`, `부서`, `직급`, `유형`)은 유지
4. 전체 테이블 최소 너비를 줄여 가로 스크롤 의존도를 낮춤

## UI 기준
- 각 칸은 `라벨 + 체크박스` 조합 유지
- 기존 색상 톤(달력정보의 인디고 계열)은 유지
- 대표/팀리더/파트리더 행 강조 색상도 유지

## 비범위
- 권한 의미 변경 없음
- 저장 구조 변경 없음
- 새 권한 추가 없음

## 완료 기준
- `달력정보` 6개 항목이 기본 화면 폭에서 한눈에 보임
- 가로 스크롤 없이 체크 상태 확인 가능
- 체크박스 저장/로딩 동작은 기존과 동일
