# 모바일 상단바 정렬 개선 계획

## 1. 목적

- 모바일에서 상단바 버튼들이 줄바꿈만 된 채 흩어져 보이는 문제를 정리한다.
- 프로필 정보와 관리자 버튼 영역을 분리해서, 휴대폰 폭에서도 읽기 쉽게 만든다.

쉽게 비유하면:
- 지금은 안내데스크 위에 명찰, 안내문, 버튼을 한꺼번에 올려둔 상태다.
- 이번 작업은 `명찰 줄`과 `버튼 줄`을 따로 만들어 정돈하는 것이다.

## 2. 현재 문제

- 모바일에서 `profileHtml`과 액션 버튼이 같은 흐름 안에 있다.
- 버튼 수가 많아지면 `flex-wrap`으로만 처리돼서, 줄이 바뀌긴 하지만 보기 좋게 정렬되진 않는다.
- 특히 관리자 계정은 버튼이 많아 더 복잡해 보인다.

## 3. 개선 방향

### 추천안

- 상단바를 모바일에서 아래 두 영역으로 분리
  1. 프로필 줄
  2. 액션 버튼 줄

### 버튼 줄 구성

- 버튼은 `2열 그리드` 형태로 정렬
- 각 버튼은 동일 높이/너비 느낌으로 맞춤
- 버튼 문구는 유지

즉:
- 위 줄: 사용자 정보
- 아래 줄: 기능 버튼들

## 4. 적용 범위

- 모바일 뷰포트(`app.isMobileViewport === true`)에서만 적용
- 데스크탑 상단바는 그대로 유지

## 5. 기대 효과

1. 버튼이 덜 흩어져 보인다.
2. 상단바 첫 인상이 가벼워진다.
3. 관리자 메뉴가 모바일에서도 누르기 쉬워진다.

## 6. 검증 계획

1. 모바일 폭에서 상단바 렌더링 확인
2. 프로필이 위쪽 한 줄로 분리되는지 확인
3. 버튼이 아래쪽 그리드로 정렬되는지 확인
4. 줄바꿈이 어색하게 깨지지 않는지 확인
5. 버튼 클릭 동작이 그대로 유지되는지 확인

## 7. 완료 기준

1. 모바일 상단바가 `프로필 줄 + 버튼 줄`로 분리된다.
2. 버튼 정렬이 지금보다 더 안정적으로 보인다.
3. 데스크탑 상단바는 영향 없다.

## 8. 진행 메모

- 2026-03-26 기준 1차 실험 적용 후 로컬 확인 결과, 버튼 영역만 일부 정리됐고 상단바 전체 개선 수준에는 미달했다고 판단했다.
- 따라서 현재는 **실험본을 원상 복구**하고,
- 이후 `로컬에서 충분히 완성한 뒤 운영 웹에 반영` 원칙으로 계속 진행한다.
