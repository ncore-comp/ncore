# 근무시간표기 / 저장검증 / 공휴일동기화 계획

## 목적
- 직원 카드의 분기 근무시간 표기를 현재 데이터 구조에 맞게 정리한다.
- 신규 직원 저장 시 분기 근무시간(Q1~Q4) 누락을 막는다.
- 관리자가 직접 공휴일 자동 동기화를 실행할 수 있는 버튼을 운영실에 추가한다.

## 쉬운 비유
- `년도 제거`는 표지판의 잘못된 연도를 지우는 작업이다.
- `Q1~Q4 저장 검증`은 접수창구에서 서류 빈칸이 있으면 접수를 안 받는 규칙을 넣는 일이다.
- `공휴일 수동 동기화 버튼`은 관리실에 `달력 새로고침 버튼`을 다는 일이다.

## 작업 1. 직원 카드 근무시간 표기에서 년도 제거

### 현재 상태
- 직원 카드에 `2026년 1분기 근무시간`처럼 표시된다.
- 실제 구조는 특정 연도 전용이 아니라, 매년 반복 사용하는 `1분기 ~ 4분기 근무시간`이다.

### 변경 방향
- 아래 문구를 모두 연도 없는 형식으로 통일
  - `1분기 근무시간`
  - `2분기 근무시간`
  - `3분기 근무시간`
  - `4분기 근무시간`

### 예상 수정 위치
- 직원 카드 렌더링
- 필요 시 구성원 관리 요약 문구

### 완료 기준
- 직원 카드에서 더 이상 `2026년` 같은 연도 표현이 보이지 않는다.

## 작업 2. 신규 직원 저장 시 Q1~Q4 미입력 차단

### 현재 상태
- 신규 직원 저장 시 `workQ1 ~ workQ4` 중 일부가 비어 있어도 저장될 수 있다.

### 변경 방향
- 신규 저장 버튼 누를 때 아래 조건을 검사한다.
  - `workQ1` 입력 여부
  - `workQ2` 입력 여부
  - `workQ3` 입력 여부
  - `workQ4` 입력 여부
- 하나라도 비어 있으면 저장 차단
- 안내 문구 예시:
  - `근무시간 분기(Q1~Q4) 중 미등록 항목이 있습니다. 모두 입력 후 저장해 주세요.`

### 원칙
- 1차 적용 범위는 `신규 저장`만
- 이메일 / 아이디 누락과 같은 수준으로 즉시 차단

### 예상 수정 위치
- 구성원 관리의 신규 직원 저장 함수

### 완료 기준
- Q1~Q4 중 하나라도 빠진 신규 직원은 저장되지 않는다.
- 경고 메시지가 즉시 표시된다.

## 작업 3. 관리자용 공휴일 수동 동기화 버튼 추가

### 현재 상태
- 대한민국 공휴일 자동 동기화는 스크립트로만 실행된다.
- 관리자가 웹에서 직접 다시 동기화할 수 있는 버튼은 없다.

### 변경 방향
- 운영실 또는 휴일 관리 섹션에 `공휴일 동기화` 버튼 추가
- 클릭 시 서버 액션을 호출해 공공데이터 공휴일 동기화 실행
- 실행 결과를 사용자에게 표시
  - 성공: 몇 건 반영됐는지
  - 실패: 인증키/원천 API 문제 안내

### 세부 원칙
- 권한은 마스터/관리자만
- 회사 수동 휴일(`manual`)은 유지
- 자동 공휴일(`auto`)과 창립기념일(`company_fixed`)은 정책대로 병합
- 실행 시 로그 남김
  - 예: `ManualHolidaySync`

### 예상 수정 위치
- 운영실 UI
- 프런트 API 함수
- functions 액션/서비스

### 완료 기준
- 관리자 화면에서 버튼 클릭으로 공휴일 동기화를 실행할 수 있다.
- 성공/실패 결과 메시지가 나온다.
- 로그가 남는다.

## 구현 순서
1. 직원 카드 분기 근무시간 표기에서 연도 제거
2. 신규 직원 저장 시 Q1~Q4 누락 차단
3. 관리자용 공휴일 수동 동기화 버튼 추가
4. `functions + hosting` 배포
5. 실제 운영 화면에서 검증

## 검증 계획

### 1. 표기 검증
- 직원 카드 열기
- 분기 근무시간 라벨에 연도 표기 없는지 확인

### 2. 신규 저장 검증
- Q1~Q4 중 하나 비운 상태로 신규 직원 저장 시도
- 저장 차단과 안내 문구 확인
- Q1~Q4 모두 채우면 정상 저장 확인

### 3. 공휴일 동기화 검증
- 관리자 화면에서 수동 동기화 버튼 클릭
- 성공/실패 메시지 확인
- Firestore `holidays`와 달력 반영 확인
