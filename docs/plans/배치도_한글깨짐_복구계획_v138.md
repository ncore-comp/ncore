# 배치도/한글깨짐 복구 계획 v138
# Seat Map / Korean Text Recovery Plan v138

## 1. 문제 요약
## 1. Issue Summary

- 해결된 문제
  - `당일 부재 인원` 아코디언 자체는 동작하게 만든 상태였습니다.
- 지금 다룰 문제
  - 전체 배치도 레이아웃이 무너졌고, 모달 내부 한글도 다시 깨진 상태입니다.
- 이번 단계에서 건드리지 않을 문제
  - 권한/설정, 운영실, 잔업/특근은 이번 복구 범위에서 제외합니다.

쉽게 비유하면,
달력 모달은 원래 `기본 건물(ui_situation_v133)` 위에
`임시 증축물(v135, v136, v137)`을 계속 얹은 상태입니다.
지금은 증축물이 겹치면서
- 간판(한글)이 깨지고
- 방 배치(레이아웃)가 어긋난 상황입니다.

In simple terms,
the seat-map modal currently runs on top of the original building (`ui_situation_v133`)
with multiple temporary extensions (`v135`, `v136`, `v137`) stacked on top.
Those layered overrides caused:
- broken Korean labels
- unstable layout

## 2. 현재 확인된 원인
## 2. Confirmed Causes

### 원인 1. `ui_situation` 오버라이드가 연속으로 겹쳐 로드되고 있음
### Cause 1. Multiple `ui_situation` override files are stacked

현재 [index.html](/D:/MYDOC/Desktop/vscode/Web/firebase/ncore_web/ncore-main/index.html) 에서는 아래가 순서대로 같이 로드됩니다.

- `ui_situation_v133_codex.js`
- `ui_situation_v135_codex.js`
- `ui_situation_v136_codex.js`
- `ui_situation_v137_codex.js`

즉 `v133` 기본 함수 위에
`v135 -> v136 -> v137` 이 순서대로 같은 함수를 덮어쓰고 있습니다.

This means the same functions are being overwritten in sequence by `v135 -> v136 -> v137`.

### 원인 2. 부분 오버라이드 파일에 깨진 한글 문자열이 들어감
### Cause 2. The partial override files contain mojibake Korean text

[ui_situation_v137_codex.js](/D:/MYDOC/Desktop/vscode/Web/firebase/ncore_web/ncore-main/js/ui_situation_v137_codex.js) 에서 확인된 예:

- `?⑥쓬`
- `?醫롮? ?醫뤾문`
- `?諭???봔???紐꾩뜚`

이 문자열들은 실제 모달 렌더 함수 안에 들어 있기 때문에,
현재 화면에 그대로 깨져서 보입니다.

These broken Korean strings are inside the live render function, so the UI shows them as-is.

### 원인 3. 레이아웃도 부분 복사 방식으로 계속 덮어써져 안정성이 떨어짐
### Cause 3. The layout is also repeatedly replaced by copied render blocks

`renderSituationBoardSeatMapModalContent()`가
`v135`, `v136`, `v137`에서 계속 부분 복사/수정됐습니다.

그래서 지금 구조는:
- 어떤 파일이 최종 기준인지 불명확
- 한글/배치도/카드 스타일이 한 함수 안에서 같이 흔들림

The modal render function was repeatedly copied and modified across `v135`, `v136`, and `v137`,
so the final source of truth is no longer clear.

## 3. 복구 원칙
## 3. Recovery Principles

1. 오버라이드 파일을 계속 누적하지 않는다.
   Do not keep stacking more override files.

2. 먼저 안정 기준선으로 되돌린다.
   First return to a stable baseline.

3. 그 다음 필요한 기능만 한 단계씩 다시 얹는다.
   Then reapply only the needed changes step by step.

4. 배치도와 한글은 같은 단계에서 함께 검증한다.
   Validate layout and Korean text together in the same step.

## 4. 권장 복구 방향
## 4. Recommended Recovery Direction

### 추천
### Recommended

`v133`을 임시 안정 기준선으로 복귀한 뒤,
새 단일 파일 `ui_situation_v138_codex.js` 하나에서만
필요한 변경을 다시 얹습니다.

Recommended approach:
temporarily return to the `v133` baseline,
then rebuild the needed changes in one single file:
`ui_situation_v138_codex.js`

### 하지 않을 것
### What we should not do

- `v138`, `v139`, `v140` 식으로 또 오버라이드 파일을 누적
- 깨진 문자열을 현재 `v137` 위에서 부분 교체
- 레이아웃과 카드 스타일을 분리하지 않고 한 번에 큰 복붙

Avoid:
- stacking more override files
- patching broken Korean directly on top of `v137`
- doing another large copy-paste rewrite without separation
