# 지난 신청 상세사유 계획

## 목적
- 지난 날짜 연차 신청과 이미 지난 시간대의 시간차 신청에 대해 상세사유를 필수로 받는다.
- 기존 반려 사유 팝업 UI를 재사용하되, 저장 필드는 `rejectReason`과 분리한다.

## 비유
- 평소 신청은 일반 신청서다.
- 지난 신청은 일반 신청서에 `지연 제출 사유서` 한 장을 더 붙이는 방식이다.

## 적용 범위
1. 시작일이 오늘보다 이전인 신청
2. 오늘 날짜의 `시간차(퇴근)` 중 시작 시간이 현재 시각보다 이전인 신청
3. 오늘 날짜의 `시간차(외출)` 중 시작 시간이 현재 시각보다 이전인 신청

## 구현 원칙
1. 기존 반려 사유 팝업을 공용 사유 팝업으로 확장
2. 저장 필드는 `detailReason` 사용
3. 반려 사유(`rejectReason`)와 의미를 섞지 않음
4. 과거 신청이 아니면 `detailReason`은 비움

## 수정 파일
- `index.html`
- `js/ui_logic_new_design_v114_codex.js`
- `js/core_logic_new_design_v114_codex.js`
- `functions/src/policies/requestPolicy.js`

## 검증 계획
1. 일반 미래 연차 신청은 추가 팝업 없이 그대로 저장
2. 지난 날짜 연차 신청은 상세사유 팝업이 뜸
3. 오늘 기준 이미 지난 시간의 시간차 신청은 상세사유 팝업이 뜸
4. 상세사유 없이 확인하면 저장되지 않음
5. 저장 후 Firestore request에 `detailReason`이 기록됨
