# 전체상황판 세로 스크롤 v139 계획
# Situation Board Vertical Scroll v139 Plan

## 한글
- 작업 번호: 2번 문제
- 현재 상태:
  - `전체 상황판` 메인에 의미 없는 세로 스크롤바가 표시됨
- 확인한 원인:
  - `main#app-container`는 `padding-top: 5.8rem`을 사용 중
  - `.situation-board-page`는 상하 패딩 `1.8rem`을 사용 중
  - 그런데 `.situation-board-shell` 높이는 `calc(100vh - 6.2rem)`로만 계산되어, page 패딩까지 반영하지 못함
  - 그 결과 문서 높이가 뷰포트보다 약 `22px` 커져 세로 스크롤 발생
- 수정 원칙:
  - `main#app-container`는 건드리지 않음
  - 가장 작은 수정으로 `.situation-board-shell` 높이 계산만 보정
- 수정 내용:
  - `.situation-board-shell`
    - `height: calc(100vh - 6.2rem)` -> `calc(100vh - 7.6rem)`
    - `min-height: calc(100vh - 6.2rem)` -> `calc(100vh - 7.6rem)`
- 기대 효과:
  - 상황판 메인이 뷰포트 높이에 정확히 맞아짐
  - 의미 없는 세로 스크롤바 제거

## English
- Issue: #2
- Current state:
  - the main situation board shows a meaningless vertical scrollbar
- Confirmed cause:
  - `main#app-container` uses `padding-top: 5.8rem`
  - `.situation-board-page` adds `1.8rem` of vertical padding
  - but `.situation-board-shell` only uses `calc(100vh - 6.2rem)`
  - so the page ends up roughly `22px` taller than the viewport
- Fix principle:
  - do not touch `main#app-container`
  - make the smallest safe fix by adjusting only the shell height calculation
- Planned change:
  - `.situation-board-shell`
    - `height: calc(100vh - 6.2rem)` -> `calc(100vh - 7.6rem)`
    - `min-height: calc(100vh - 6.2rem)` -> `calc(100vh - 7.6rem)`
- Expected result:
  - the situation board main area fits the viewport exactly
  - the meaningless vertical scrollbar disappears
