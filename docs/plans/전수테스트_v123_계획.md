## 테스트 기본 원칙
## Test Principles
1. 먼저 `로그인 -> 홈 대시보드`가 정상인지 확인합니다.
1. First verify `login -> home dashboard`.
2. 그다음 `권한/설정 -> 운영실 -> 구성원 관리`처럼 관리축을 확인합니다.
2. Then verify the admin flows such as `permissions -> admin ops -> user management`.
3. 이후 `연차`, `잔업/특근`, `게시판`, `전체 상황판` 순으로 기능축을 확인합니다.
3. Then verify feature flows: `leave`, `work report`, `board`, `situation board`.
4. 전수 테스트 중 오류가 나오면 **새 기능 추가 없이 원인만 고정**하고 중단합니다.
4. If an error appears, stop and record the cause without adding new feature work.

## 결과 기록 방식
## Result Recording Format
- 각 항목은 아래 3단계로 기록합니다.
- Record each item with these three states.
  - 소스 파일 수정 여부
  - Whether source code was changed
  - 현재 운영 반영 여부
  - Whether it is live in production
  - 사용자 화면 확인 여부
  - Whether it was confirmed on the user-visible screen

- 테스트 성공은 아래 3종을 분리합니다.
- Separate test success into these three types.
  - 단독 함수 테스트 성공
  - Single-function test passed
  - 통합 경로 테스트 성공
  - Integrated flow test passed
  - 실제 사용자 실행 경로 성공
  - Real user execution path passed
