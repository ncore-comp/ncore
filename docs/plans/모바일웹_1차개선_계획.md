# 모바일웹 1차 개선 계획

## 1. 방향

- 현재 상태는 `모바일 대응 분기 코드가 있음`이지, `모바일 최적화 완료`는 아니다.
- 따라서 지금은 PWA보다 먼저 **모바일 웹 사용성 개선**을 1차로 진행한다.

쉽게 비유하면:
- 앱 간판을 달기 전에
- 휴대폰 손님이 매장 안에서 불편하지 않게 통로와 진열대를 먼저 정리하는 단계다.

## 2. 목표

1. 휴대폰에서 핵심 업무 흐름이 막히지 않게 한다.
2. 버튼/모달/달력/표가 손가락으로 쓰기 쉽게 정리되게 한다.
3. 이후 PWA를 붙여도 무리 없는 상태를 만든다.

## 3. 1차 점검 대상 화면

우선순위가 높은 순서대로 보면 아래다.

### 1순위

1. 로그인 화면
2. 대시보드
3. 연차 신청 모달
4. 최근 신청 내역

### 2순위

5. 전체상황판
6. 운영실
7. 구성원 관리

### 3순위

8. 권한/설정
9. 게시판
10. 직원 카드

## 4. 모바일에서 예상되는 문제

현재 구조를 보면 아래 문제가 나올 가능성이 높다.

1. 버튼이 작거나 너무 붙어 있음
2. 모달 높이가 길어서 하단 버튼이 잘 안 보임
3. 권한표 / 운영실 표 / 설정 표가 가로로 넘침
4. 달력 칸이 좁아져서 정보가 겹침
5. 네비 버튼이 많아서 상단이 답답함
6. 툴팁/hover 중심 요소는 모바일에서 의미가 약함
7. CSV 다운로드 / 메일 호출이 모바일에서 덜 자연스러울 수 있음

## 5. 1차 개선 범위

이번 1차는 `핵심 업무 흐름이 끊기지 않게 하는 것`에 집중한다.

### 포함

1. 상단 네비 간격/줄바꿈 정리
2. 연차 신청 모달 세로 스크롤 정리
3. 버튼 터치 영역 확대
4. 대시보드 카드 레이아웃 정리
5. 운영실 주요 카드 세로 흐름 점검
6. 전체상황판 모바일 표시 가독성 점검

### 제외

1. 모든 화면의 완전 재설계
2. 앱 수준 제스처 설계
3. 네이티브 기능 대응

즉:
- 1차는 `쓸 수 있게 만들기`
- 2차부터 `더 좋게 만들기`

## 6. 세부 체크리스트

### 로그인

- 입력칸 크기 충분한가
- 버튼 터치 쉽나
- 키보드 올라와도 버튼 안 가려지나

### 대시보드

- 상단 요약 카드가 너무 좁지 않나
- 최근 신청 내역 카드가 읽기 쉬운가
- 연차 신청 버튼이 바로 눌리나

### 연차 신청 모달

- 모달이 화면을 넘치지 않나
- 종류/날짜/사유 선택이 손가락으로 편한가
- 하단 저장 버튼이 항상 보이거나 쉽게 스크롤 가능한가

### 전체상황판

- 월간 칩이 겹치지 않나
- 달력 셀이 너무 빽빽하지 않나
- 자리배치도 모달이 모바일에서 지나치게 넓지 않나

### 운영실

- 1열 세로 배치가 모바일에서도 자연스러운가
- 대상 직원 선택 / 대리 등록 버튼이 눌리기 쉬운가
- 최근 로그 카드가 너무 길거나 답답하지 않나

### 구성원 관리 / 권한표

- 표가 너무 넓으면 가로 스크롤 유도 필요
- 버튼/체크박스 터치 영역이 충분한지 확인

## 7. 진행 순서

### 1단계

- 모바일 점검 기준 문서 확정

### 2단계

- 실제 모바일 뷰포트 기준 스크린샷/검증

### 3단계

- 1순위 화면부터 수정

### 4단계

- 운영 반영 후 재검증

### 5단계

- 결과 문서 정리

## 8. 완료 기준

아래가 되면 1차 개선 완료로 본다.

1. 로그인 / 대시보드 / 신청 모달이 모바일에서 무리 없이 사용된다.
2. 운영실과 전체상황판이 최소한 `보기/조작` 가능한 수준이다.
3. 모바일에서 명백히 깨지는 레이아웃이 없다.
4. 이후 PWA 작업으로 넘어가도 되는 상태다.

4. PWA 도입 시작

## 10. 결론

지금은 PWA를 바로 붙이는 것보다,  
**모바일 웹을 먼저 “폰에서 실제로 쓸 수 있는 상태”로 만드는 것이 우선**이다.

즉 이번 단계는
- 앱 포장보다
- 모바일 사용성 정리
를 먼저 하는 단계다.
