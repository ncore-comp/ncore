## 한 줄 요약

- 잔업/특근은 `신청`과 `현황`을 분리합니다.
- 직원은 `잔업/특근 신청`에서 자기 건을 기록, 수정, 제출합니다.
- 승인권자는 `잔업/특근 현황`에서 제출 묶음을 보고 상세 확인 후 승인/반려합니다.
- 승인권자는 연차 승인권자와 동일합니다.

쉽게 비유하면:
- `잔업/특근 신청` = 직원이 영수증을 적는 창구
- `잔업/특근 현황` = 승인권자와 관리자가 장부를 보는 열람실
- `제출` = 영수증을 봉투로 묶어 결재 라인에 올리는 것

## 확정 규칙

### 1. 승인권자
- 잔업/특근 승인권자는 기존 연차 승인권자와 동일합니다.
- `approveScope`를 그대로 재사용합니다.

### 2. 버튼 구조
- 메인 화면 버튼:
  - `잔업/특근 신청`
  - `잔업/특근 현황`

### 3. 대표 예외
- 대표는 `잔업/특근 신청` 버튼 없음
- 대표는 `잔업/특근 현황`만 사용

### 4. 팀장 예외
- 팀장은 승인권자이면서 자기 잔업/특근도 신청 가능
- 연차 규칙과 동일하게 해석

### 5. 제출 전/후 수정 규칙
- 제출 전: 자유 수정 가능
- 제출 후: 수정 불가
- 반려 후: 다시 수정 가능
- 수정 후: 재제출 필요

## 권한 구조

### 1. 잔업/특근 권한 그룹
권한/설정에 `잔업/특근` 그룹 추가

권장 항목:
- `잔업/특근 신청`
- `잔업/특근 현황(매뉴얼팀)`
- `잔업/특근 현황(파츠북팀)`
- `잔업/특근 현황(모두)`

### 2. 현황 권한 자동 부여
- 현황 권한은 체크박스로 별도 부여 가능
- 승인권자는 같은 범위의 현황 권한을 자동으로 가짐

예:
- `approveScope = manual` -> `현황(매뉴얼팀)` 자동
- `approveScope = parts` -> `현황(파츠북팀)` 자동
- `approveScope = all` -> `현황(모두)` 자동

## 설정 위치

- 권한/설정 상단에 `잔업/특근 설정` 탭 추가
- 위치는 `운영실` 왼쪽

예상 순서:
- 권한 탭
- 설정 테이블
- 메일 설정
- 잔업/특근 설정
- 운영실

## 달력 표시 규칙

### 1. 개인 탭
- 달력 `개인` 탭에는 아래 두 가지를 함께 표시
  - 내 연차
  - 내 잔업/특근

### 2. 잔업/특근 탭
- 달력에 별도 `잔업/특근` 탭을 둠
- 관련 권한이 있는 사람만 노출
- 현황 권한 범위에 따라 매뉴얼팀/파츠북팀/모두 열람 가능

## 신청 화면

`잔업/특근 신청` 버튼 진입 화면

구성:
- 내 건만 표시
- 건별 수정 버튼
- 정산 제출 전 요약 확인
- `승인 요청` 버튼

규칙:
- 제출 전 수정 가능
- 제출 후 잠금
- 반려 후 다시 수정 가능

## 현황 화면

`잔업/특근 현황` 버튼 진입 화면

구성:
- 권한 범위 직원들의 정산월 집계표
- 직원별 요약
- `상세확인`

승인권자는 상세 화면에서:
- `승인`
- `반려`

## 홈 결재 대기 카드

- 휴가 승인 대기와 잔업/특근 제출 대기를 같은 카드 안에 둠
- 가로 구분선으로 섹션 분리

구성:
- 위: 기존 휴가 승인 대기
- 구분선
- 아래: 잔업/특근 제출 대기

잔업/특근 항목에는 홈에서 `상세확인`만 둠
- 홈에서 `반려` 버튼은 만들지 않음
- 승인/반려는 상세 화면 상단에서 처리

## 메일 흐름

### 1. 직원 -> 승인권자
- 직원이 승인 요청 시 일반 연차와 동일한 메일 설정 재사용
- 메일에는 요약만 담음
  - 정산기간
  - 총 잔업시간
  - 총 특근일수
  - 상세는 웹에서 확인

### 2. 승인권자 -> 직원
- 승인 또는 반려 시 요청자에게 결과 메일 발송
- 반려 시 반려 사유 포함

## 데이터 구조 권장

- 건별 요청은 기존 `requests` 유지
- 정산 묶음 컬렉션 도입
  - 예: `workReportSettlements`

역할:
- 정산기간 단위 묶음
- 제출 상태
- 승인 상태
- 승인자 정보
- 반려 사유
- 메일 발송 이력

쉽게 비유하면:
- `requests` = 영수증 한 장
- `workReportSettlements` = 영수증 봉투

## 구현 순서

1. 권한/설정 구조 추가
2. `잔업/특근 신청` / `잔업/특근 현황` 버튼 분리
3. 신청 화면 / 현황 화면 분리
4. 달력 개인 탭 통합 표시
5. 달력 `잔업/특근` 탭 권한 연동
6. 정산 묶음 생성 및 승인 요청
7. 홈 결재 대기 카드 분리
8. 상세 화면 승인/반려
9. 결과 메일 발송

## 주의할 점

- 제출 후 수정 잠금이 확실해야 함
- 반려되면 수정 가능 상태로 다시 열려야 함
- 승인권자 범위와 현황 범위 충돌 방지
- 대표는 신청 화면 진입이 없어야 함
