# 잔업특근 신청창 통합개선 계획

## 작업 일시
- 2026-04-01

## 작업 배경
- 현재 메인 화면에는 `잔업 신청`, `특근 신청` 버튼이 따로 존재합니다.
- 사용자는 두 버튼이 따로 나뉘어 있는 대신, 하나의 `잔업/특근신청` 버튼으로 통합되길 원합니다.
- 또한 현재 `보고 추가` 흐름은 결국 기존 `연차 신청` 모달을 재사용하면서, 종류 드롭다운에 `잔업 보고`, `특근 보고`를 섞어 넣는 방식입니다.
- 이 구조는 사용자 입장에서 `연차 신청`과 `잔업/특근 보고`의 성격이 다름에도 한 창구에서 섞여 보여 혼란을 줍니다.

## 핵심 문제 정의
1. 진입 버튼이 기능별로 쪼개져 있어 사용자가 어디를 눌러야 할지 직관적이지 않음
2. 잔업/특근 보고가 연차 신청 모달 안에 들어가 있어 기능의 의미가 흐려짐
3. 연차 신청과 잔업/특근 보고는 승인 흐름, 차감 구조, 입력 항목이 다르므로 UI도 분리되는 것이 자연스러움

## 목표
- 메인 화면의 `잔업 신청`, `특근 신청` 버튼을 하나의 `잔업/특근신청` 버튼으로 통합
- 이 버튼을 누르면 `연차 신청` 모달이 아니라 **별도의 잔업/특근 전용 신청 모달**이 뜨도록 변경
- 기존 월별 잔업/특근 현황판과 자연스럽게 이어지도록 진입 흐름 재정리

## 추천 방향
### 1. 버튼 구조
- 메인 화면 퀵액션 영역에서
  - 기존 `잔업 신청`
  - 기존 `특근 신청`
  를 제거
- 대신 하나의 통합 버튼 추가
  - `잔업/특근신청`

### 2. 버튼 노출 규칙
- `잔업 신청 기능` 또는 `특근 신청 기능` 둘 중 하나라도 켜져 있으면 통합 버튼 표시
- 둘 다 꺼져 있으면 버튼 숨김

### 3. 신청창 구조
- 통합 버튼 클릭 시 **잔업/특근 전용 신청 모달** 오픈
- 연차 신청 모달(`연차/반차/시간차`)과는 완전히 분리
- 모달 내부 첫 항목은 `구분`
  - `잔업`
  - `특근`
- 여기서 선택한 값에 따라 아래 입력 규칙과 안내문이 달라짐

## 신청창 상세 설계
### 공통 입력 항목
- 구분: `잔업`, `특근`
- 날짜
- 시작시간
- 보고시간
- 사유
- 업무내용
- 요청 부서
- 비고

### 잔업 규칙
- 평일 기준 보고
- 저장된 퇴근시간을 기준으로 1시간 단위 신청
- 특근이 없는 일반 평일 잔업은 기존 규칙 유지
- 사용자가 지난 날짜 선택 시에도 현재 허용 정책 유지

### 특근 규칙
- 휴일/주말/회사 휴무일에도 가능
- 시작시간과 보고시간을 직접 입력
- 1시간 단위 보고

### 연동 규칙
- 같은 날짜에 `특근 + 잔업` 동시 존재 가능
- 잔업/특근은 승인 없는 `reported` 흐름 유지
- 연차 차감, 승인 대기, 부재 달력 차감 흐름과 분리

## 현황판과의 연결 방식
### 추천안
- `잔업/특근신청` 버튼 클릭 시 바로 신청 모달을 열지 않고,
- 먼저 `정산월 잔업/특근 현황` 모달을 열고,
- 그 안의 `보고 추가` 버튼으로 전용 신청 모달을 열도록 유지/정리

### 이유
- 사용자는 먼저 장부를 보고 현재 월 기록을 확인할 수 있음
- 그 다음 필요한 경우에만 새 보고를 추가
- 기존에 이미 만들어둔 `월별 잔업/특근 현황` 흐름을 재사용할 수 있음

## UI/UX 권장 방향
### 메인 버튼
- 라벨: `잔업/특근신청`
- 색상은 중립 계열 유지
- 아이콘은 시계 또는 문서 아이콘 사용

### 전용 신청 모달
- 제목:
  - 기본: `잔업/특근 신청`
  - 내부 `구분` 선택으로 세부 상태 전환
- 보조 문구:
  - `잔업/특근은 보고용이며 승인 절차 없이 저장됩니다.`
- 구분 선택 시 안내 문구 변경
  - `잔업`: 평일 연장근무 보고
  - `특근`: 휴일/주말 근무 보고

## 구현 범위
- 메인 화면 퀵액션 버튼 구조 변경
- 잔업/특근 버튼 노출 조건 통합
- 별도 `잔업/특근 신청` 모달 추가
- 기존 `연차 신청` 모달에서 `잔업 보고`, `특근 보고` 옵션 제거
- `보고 추가` 버튼이 새 전용 모달을 열도록 변경
- 기존 저장 API/데이터 구조는 최대한 재사용

## 데이터 구조 방향
- 기존 `requests` 저장 구조 유지
- 계속 사용하는 필드
  - `reportCategory`
  - `workDetail`
  - `requestDept`
  - `note`
  - `requestedStartAt`
  - `requestedEndAt`
  - `reportedHours`
- 즉 저장 구조를 다시 만드는 것이 아니라 **입력 UI만 분리**하는 방향

## 기대 효과
- 연차 신청과 잔업/특근 보고의 역할 분리
- 사용자 입장에서 기능 이해가 쉬워짐
- 버튼 수는 줄고 의미는 더 선명해짐
- 현재 만든 잔업/특근 현황판과 신청 흐름이 더 자연스럽게 연결됨

## 구현 순서 제안
1. 메인 버튼 2개를 `잔업/특근신청` 1개로 통합
2. 통합 버튼 클릭 시 현황판 오픈 구조 정리
3. 별도 `잔업/특근 신청` 모달 추가
4. 기존 `연차 신청` 모달의 잔업/특근 옵션 제거
5. `보고 추가` 버튼을 새 전용 모달에 연결
6. 저장/검증/문구/안내 정리

## 완료 기준
- 메인 화면에 `잔업/특근신청` 버튼 1개만 보임
- 잔업/특근 보고는 더 이상 `연차 신청` 모달에서 하지 않음
- `잔업/특근 신청` 전용 모달에서만 보고 가능
- 기존 월별 현황판과 연결이 자연스럽게 이어짐
