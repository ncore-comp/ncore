## 작업 원칙
## Working Rules

1. 새 코드 수정은 항상 `ncore-main`에서만 한다.
1. Make new code changes only inside `ncore-main`.

2. `legacy` 폴더는 참고 전용으로 보고, 현재 배포본처럼 수정하지 않는다.
2. Treat `legacy` as reference-only and do not edit it like the current deployable source.

3. 문서 신규 작성은 가능하면 `docs` 아래 분류 폴더에 둔다.
3. Place new documents under the appropriate category inside `docs` whenever possible.

4. `handoff.md`, `이후_작업_정리.md`는 루트 유지 문서로 계속 사용한다.
4. Keep using `handoff.md` and `이후_작업_정리.md` as root-level living documents.

5. GitHub 업로드가 필요하면 원본을 정리하지 말고 `github_upload_ready_*` 패키지를 새로 만든다.
5. When GitHub upload is needed, do not rearrange the active source; create a fresh `github_upload_ready_*` package instead.
