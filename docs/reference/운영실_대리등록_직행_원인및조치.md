# 운영실 대리등록 결재대기 원인 및 조치

업데이트: 2026-03-25

## 1. 문제

운영실에서 관리자가 다른 직원 연차를 대리 등록했는데,
해당 건이 즉시 반영되지 않고 `결재 대기` 목록으로 들어갔다.

즉 운영실 등록이 `관리자 직행 입력`처럼 보여도,
실제로는 `일반 신청서 접수`처럼 저장되고 있었다.

## 2. 원인

### 프런트 원인

신규 등록 코드에서 관리자 대리 등록 여부와 상관없이
새 요청을 항상 `status: pending`으로 생성하고 있었다.

대상 파일:

- [js/ui_logic_new_design_v114_codex.js](/D:/MYDOC/Desktop/vscode/Web/firebase/ncore_web/ncore-main/js/ui_logic_new_design_v114_codex.js)

### 서버 원인

서버는 신규 요청 시

- 일반 자기 신청
- 관리자 대리 등록

을 구분하지 않고 모두 `create`로 처리하고 있었다.

즉 `관리자 직행 입력`이라는 별도 개념이 없었다.

대상 파일:

- [functions/src/policies/requestPolicy.js](/D:/MYDOC/Desktop/vscode/Web/firebase/ncore_web/ncore-main/functions/src/policies/requestPolicy.js)
- [functions/src/services/requestService.js](/D:/MYDOC/Desktop/vscode/Web/firebase/ncore_web/ncore-main/functions/src/services/requestService.js)

## 3. 조치 방향

### 적용안

`운영실 등록 = 즉시 확정 입력`

규칙:

- 저장 시 `approved`
- 결재 대기 미노출
- 메일 미발송
- 로그는 `AdminRequestCreate`

쉽게 비유하면:

- 운영실 등록은 접수창구가 아니라
- 관리자가 장부에 바로 적는 직행 통로로 바꾼다

## 4. 구현 원칙

1. 프런트에서 운영실 신규 등록 시 `approved`로 생성
2. 서버에서 관리자 신규 등록을 `admin_create`로 별도 구분
3. `admin_create`는 저장 시 `approved`
4. 로그는 `AdminRequestCreate`
5. 메일 발송 없음

## 5. 완료 기준

1. 운영실 등록 건이 결재 대기에 뜨지 않는다
2. DB에 바로 `approved`로 저장된다
3. 메일이 발송되지 않는다
4. `AdminRequestCreate` 로그가 남는다
