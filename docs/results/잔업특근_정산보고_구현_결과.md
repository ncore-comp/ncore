## 이번에 구현한 범위

### 1. 정산 제출 API 추가

- 새 action: `submit_work_report_settlement`
- 대상:
  - 로그인한 사용자의 현재 정산기간 잔업/특근 보고 건
- 저장 필드:
  - `settlementPeriodKey`
  - `settlementSubmittedAt`
  - `settlementSubmittedBy`
  - `settlementSubmittedByName`

### 2. 요청 데이터 로드 범위 보강

- 잔업/특근 보고에 필요한 아래 필드를 `load_data` 응답에 포함
  - `reportCategory`
  - `workDetail`
  - `requestDept`
  - `note`
  - `requestedStartAt`
  - `requestedEndAt`
  - `reportedHours`
  - `settlementPeriodKey`
  - `settlementSubmittedAt`
  - `settlementSubmittedBy`
  - `settlementSubmittedByName`

### 3. 개인용 정산 기능 추가

- 잔업/특근 현황판(개인용)에 아래 추가
  - `메일 초안` 버튼
  - `제출 완료` 버튼
  - 정산 제출 상태 배너
    - `미제출 n건 / 제출 예정일`
    - 또는 `정산 제출 완료 / 제출시각`

### 4. 메일 초안 생성 기능 추가

- 정산기간(`16일~다음달 15일`) 기준으로
  - 날짜별 행(row)
  - 작업시작~완료
  - 작업시간
  - 잔업/특근
  - 업무내용
  - 요청 부서
  - 비고
를 그대로 유지한 메일 본문 초안 생성

- 메일 초안 모달에서 제공:
  - 받는 사람
  - 참조
  - 제목
  - 본문
  - `본문 복사`
  - `메일 앱 열기`

### 5. 관리자용 제출 상태 표시 추가

- 관리자 집계표에 `제출 상태` 컬럼 추가
- 표시:
  - `제출 완료`
  - `미제출`
- 제출 완료 건은 최근 제출 시각도 함께 표시

### 6. 상세 화면 제출 상태 표시 추가

- 관리자 상세/개인 상세 상단에
  - 제출 완료 여부
  - 최근 제출 시각
표시
