## 반영 내용

- 홈 화면 `잔업/특근 제출 대기`가 더 이상 현재 달력 월만 보지 않도록 수정했습니다.
- 이제 `settlementStatus = submitted` 상태의 잔업/특근 정산 제출 건을 월과 상관없이 모두 모아서 표시합니다.
- `상세확인`을 누르면 해당 직원의 해당 정산월로 바로 이동해서 상세를 볼 수 있습니다.

## 원인

- 기존에는 홈 결재함이 `app.viewYear`, `app.viewMonth` 기준으로만 잔업/특근 현황을 읽고 있었습니다.
- 그래서 이전달에 승인 요청한 정산 묶음은 홈 결재함에서 누락됐습니다.

## 변경 포인트

- `getPendingWorkReportSettlementSummaries()` 헬퍼 추가
  - 전체 `requests`에서
  - `userId + settlementPeriodKey` 단위로 제출 묶음을 그룹화
  - `submitted` 상태만 홈 결재함에 노출
- 홈 결재함 카드에서
  - 현재 월 라벨 대신
  - 각 제출 묶음의 실제 정산월(`periodYear`, `periodMonth`)을 표시
- `상세확인` 클릭 시
  - 해당 정산월 기준으로 `잔업/특근 현황` 모달을 연 뒤
  - 해당 직원 상세로 진입
