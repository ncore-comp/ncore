# 게시판 권한 읽기/쓰기 v127 결과
# Board Read/Write Permission v127 Result

## 요약
## Summary
- `v127`로 버전을 올려 공지사항(기존 게시판) 권한 구조를 `전역 스위치 + 개인 읽기/쓰기 권한`으로 확장했습니다.
- The app was bumped to `v127`, and the notice-board permission model was expanded into `global switch + per-user read/write permissions`.

- 이번 단계에서는
  - 권한 모델 추가
  - 권한 탭 UI 추가
  - 프런트 버튼/글쓰기 판정 통일
  - 서버 게시글 로드/쓰기 판정 통일
  까지 반영했습니다.
- This phase includes:
  - permission model extension
  - permission-tab UI addition
  - unified frontend notice visibility / write checks
  - unified server read/write checks for board posts

## 반영 내용
## Applied Changes

### 1. 새 권한 필드
### 1. New Permission Fields
- `permissions.boardRead`
- `permissions.boardWrite`

### 2. 권한 탭 UI
### 2. Permission Tab UI
- `게시판` 그룹 추가
- Added a `게시판` permission group

- 체크박스 2개 추가
- Added 2 checkboxes
  - `읽기`
  - `쓰기`

### 3. 프런트 판정
### 3. Frontend Permission Checks
- 상단 `공지사항` 버튼은 `boardRead` 기준으로 노출
- Top-nav `공지사항` button now follows `boardRead`

- `글쓰기` 버튼은 `boardWrite` 기준으로만 노출
- The `글쓰기` button now appears only when `boardWrite` is allowed

- 수정/삭제는 `master` 또는 `boardWrite + 본인 글` 기준
- Edit/delete now follows `master` or `boardWrite + own post`

### 4. 서버 판정
### 4. Server Permission Checks
- `loadService`
  - 공지사항 로드 시 `boardRead` 기준 적용
- `loadService`
  - Board loading now respects `boardRead`

- `boardService`
  - 글쓰기/삭제 시 `boardWrite` 기준 적용
- `boardService`
  - Create/delete now respects `boardWrite`
