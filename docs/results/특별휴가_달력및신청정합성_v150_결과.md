# 특별휴가 달력 및 신청 정합성 v150
# Special Leave Calendar and Request Consistency v150

- 해결한 문제
- Resolved issues
  1. 특별휴가가 여러 건 겹칠 때 달력 막대바/발생 칩의 색이 모두 같아 구분이 어려웠습니다.
  1. When multiple special-leave grants overlapped, the calendar bars/chips used the same color and were hard to distinguish.
  2. 신청 불가 날짜인데도 신청 모달의 날짜 안내가 `신청 가능한 날짜입니다`로 먼저 보였습니다.
  2. The request modal showed `This date is available` even for dates that were invalid for the selected special leave.
  3. 선택한 날짜와 무관하게 등록된 특별휴가가 모두 신청 종류 드롭다운에 나타났습니다.
  3. The request-type dropdown showed all registered special leaves regardless of the selected date.

- 변경 내용
- Changes
  - `ui_calendar_v150_codex.js`
    - 같은 달에 활성 특별휴가 grant가 여러 건이면 grant별로 서로 다른 표시 색을 배정하도록 변경했습니다.
    - If multiple active special-leave grants exist in the month, each grant is now assigned a distinct display color.
  - `ui_shared_helpers_v150_codex.js`
    - 특별휴가 드롭다운이 선택한 날짜(`req-start-date ~ req-end-date`) 안에서 실제 사용 가능한 grant만 보여주도록 바꿨습니다.
    - The special-leave dropdown now shows only grants that are actually usable within the selected date range (`req-start-date ~ req-end-date`).
    - 여러 grant를 자동 합산하지 않고, 날짜 범위에 맞는 grant만 독립적으로 표시합니다.
    - Grants are not auto-combined; only individually valid grants are shown for the chosen date range.
  - `ui_logic_new_design_v150_codex.js`
    - 날짜 변경 시 특별휴가 옵션을 먼저 다시 계산하고,
    - When the date changes, special-leave options are recalculated first.
    - 기존에 선택한 특별휴가가 현재 날짜에 불가하면 드롭다운에서 제외하고 즉시 경고 문구를 보여주도록 변경했습니다.
    - If the previously selected special leave is invalid for the current date, it is removed from the dropdown and an immediate warning message is shown.
    - `req-type`, `req-special-mode` 변경 시에도 날짜 상태 문구를 즉시 다시 계산하도록 변경했습니다.
    - The date-status message is now recomputed immediately when `req-type` or `req-special-mode` changes.

- 기대 결과
- Expected result
  - 겹치는 특별휴가 막대바는 서로 다른 색으로 보여야 합니다.
  - Overlapping special-leave bars should now appear in different colors.
  - 신청 불가 날짜의 특별휴가는 드롭다운에 보이지 않아야 합니다.
  - Special leaves that are not valid on the selected date should not appear in the dropdown.
  - 이미 고른 특별휴가가 날짜 변경으로 불가해지면, 신청 버튼을 누르기 전에 경고 문구가 먼저 보여야 합니다.
  - If a selected special leave becomes invalid after changing the date, a warning should appear before the user presses submit.
