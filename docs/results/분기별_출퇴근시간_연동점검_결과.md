## 1. 점검 목적

직원별 `workQ1 ~ workQ4` 값이 실제로 아래 기능과 정확히 연동되는지 확인한다.

- 1분기/2분기/3분기/4분기 근무표를 벽에 붙여놓았을 때
- 창구 직원이 날짜를 보고 맞는 분기 표를 꺼내 쓰는지 확인하는 점검이다

## 3. 현재 동작 규칙

### 3-1. 분기 판정 규칙

날짜 기준 분기 키:

- 1~3월 -> `workQ1`
- 4~6월 -> `workQ2`
- 7~9월 -> `workQ3`
- 10~12월 -> `workQ4`

### 3-2. 실제 조회 규칙

`getWorkShiftForDate(user, dateStr)`는

1. 해당 분기 키
2. `workQ1`
3. `workQ2`
4. `workQ3`
5. `workQ4`
6. 기본값 `09:00 ~ 18:00`

순으로 fallback 한다.

즉:

- 해당 분기 값이 있으면 그 값을 사용
- 해당 분기 값이 비어 있으면 다른 분기 값 중 먼저 발견되는 값을 사용

## 4. 실제 운영 데이터 확인

대상 직원:

- `김효창`

운영 서버에서 확인한 값:

- `workQ1 = 07:00 ~ 16:00`
- `workQ2 = 07:00 ~ 16:00`
- `workQ3 = ''`
- `workQ4 = ''`

운영 서버 계산 결과:

- `2026-03-20` -> `workQ1` -> `07:00 ~ 16:00`
- `2026-04-01` -> `workQ2` -> `07:00 ~ 16:00`
- `2026-07-01` -> `workQ3`이 비어 있으므로 fallback -> `07:00 ~ 16:00`
- `2026-10-01` -> `workQ4`가 비어 있으므로 fallback -> `07:00 ~ 16:00`

즉 현재 김효창 데이터는

- Q1/Q2만 직접 값이 있고
- Q3/Q4는 비어 있어서
- 같은 근무시간이 fallback으로 적용되는 상태다

## 6. 저장 로직 연동 여부

### 시간차(퇴근)

저장 시:

- `submitRequest()`에서 `requestUser`
- 그리고 `getTimeoffEndHourForDate(requestUser, sDate)`
를 사용한다

즉:

- 로그인한 사용자가 아니라
- 실제 대상 직원 기준으로 종료 시간이 계산된다

### 반차

저장 시:

- `buildHalfDayTimeRange(requestUser, sDate, type)`
를 사용한다

즉:

- 반차도 대상 직원의 분기 근무시간 기준으로 시간 범위를 만든다

## 7. 운영실 대리수정 연동 여부

운영실 수정 모달에서:

- `syncRequestTypeDefaults()`가
- `app.getRequestModalTargetUser()`를 우선 사용하도록 반영돼 있다

김효창 검증 결과:

- 수정 모달 퇴근시간 표시: `16`
- 저장 후 `timeRange: 13:00~16:00`
- `status: approved` 유지

즉 운영실 대리수정도 대상 직원의 현재 분기 근무시간 기준으로 연동된다

## 8. 후속 조치

연동 점검 이후 실제 운영 데이터 보정을 실행했다.

- `workQ2`는 있고 `workQ3`, `workQ4`가 비어 있던 직원 전원을 대상으로
- `Q3 = Q2`, `Q4 = Q2` 보정을 완료했다

결과:

- `updatedCount: 36`
- `remainingBlankTargets: 0`

즉 이 문서에서 주의사항으로 남겨두었던
`Q3/Q4 비어 있음 -> fallback 의존`
상태는 현재 운영 데이터 기준으로 해소된 상태다.

## 9. 최종 판단

### 확인된 것

1. 분기별 날짜 판정 정상
2. 시간차(퇴근) 저장 시 분기 근무시간 반영 정상
3. 반차 계산 시 분기 근무시간 반영 정상
4. 운영실 관리자 대리수정 시 대상 직원 기준 반영 정상

### 주의할 점

현재 구조는 분기 값이 비어 있을 때 다른 분기 값으로 fallback 한다.

즉:

- `workQ3`, `workQ4`를 비워두면
- 이전 분기 값이 이어져 보일 수 있다

이건 버그라기보다 현재 설계 특성이다.

운영상 더 명확하게 하려면:

- 각 직원의 `workQ1~workQ4`를 모두 채워두는 것이 가장 안전하다

## 10. 결론

현재 코드와 운영 서버 기준으로,
분기별 출퇴근시간 데이터는 실제 연차/반차/시간차/운영실 수정에 연동되고 있다.

그리고 후속 보정까지 완료되어,
현재 운영 데이터는 `Q2만 있고 Q3/Q4가 비어 있는 상태`가 아니라
하반기 분기값까지 명시적으로 채워진 상태다.
