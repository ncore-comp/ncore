# 비밀번호 변경 재인증 결과

## 1. 목적

- 비밀번호 변경 전에 `현재 비밀번호`를 다시 확인하는 절차를 추가
- 화면과 서버가 둘 다 같은 규칙으로 움직이도록 맞춤

쉽게 말하면:
- 새 자물쇠를 달기 전에, 지금 열쇠가 진짜 본인 열쇠인지 먼저 확인하도록 바꾼 작업이다.

## 2. 반영 내용

### 2.1 비밀번호 변경창 UI

입력칸을 아래 3개로 변경:

1. 현재 비밀번호
2. 변경할 비밀번호
3. 변경할 비밀번호 재확인

비밀번호 보기 토글도 3개 입력칸 모두에 적용되도록 보정

반영 파일:

### 2.2 서버 검증

서버에서 아래를 확인하도록 변경:

1. 현재 비밀번호 입력 여부
2. 현재 비밀번호 일치 여부
3. 새 비밀번호 입력 여부
4. 새 비밀번호 길이(4자 이상)

반영 파일:

## 4. 실제 검증 결과

테스트 계정:
- `Web관리자 (id: 0)`

검증 흐름:

1. 현재 비밀번호를 틀리게 넣고 변경 시도
   - 결과: `CURRENT_PASSWORD_INVALID`
   - 정상 차단

2. 현재 비밀번호 `0`, 새 비밀번호 `0001`로 변경 시도
   - 결과: 성공

3. 이전 비밀번호 `0`으로 재로그인 시도
   - 결과: 실패

4. 새 비밀번호 `0001`로 로그인 시도
   - 결과: 성공

### 테스트 정책 복구

- 테스트 기간 정책상 전 직원 초기 비밀번호는 `0`이어야 하므로,
- 검증 후 다시 일괄 초기화 스크립트 실행

복구 결과:
- 전체 사용자 `36명`
- 초기 비밀번호 `0`
- 로그인 검증 다시 성공

## 5. 주의

- 비밀번호 변경 기능은 이제 현재 비밀번호를 알아야만 사용 가능
- 테스트 기간 중에도 새 비밀번호는 최소 4자 이상이어야 함
- 테스트 정책 유지가 필요할 경우, 개별 변경 대신 일괄 초기화 스크립트로 다시 `0` 상태를 맞출 수 있음
