# 운영실 대리등록 직행 결과

업데이트: 2026-03-25

## 1. 문제 요약

운영실에서 관리자가 다른 직원 연차를 대리 등록했을 때,
결재 메일은 가지 않는데 DB에는 `pending`으로 들어가서 `결재 대기` 목록에 노출되는 문제가 있었다.

## 2. 원인

### 프런트

운영실 신규 등록도 일반 신청과 같은 생성 레일을 타고 있었고,
새 요청을 항상 `status: pending`으로 만들고 있었다.

### 서버

서버도 신규 요청을

- 일반 자기 신청
- 관리자 대리 등록

으로 구분하지 않고 모두 `create`로 처리하고 있었다.

즉 운영실 등록이 `관리자 직행 입력`이 아니라 `대리 신청`처럼 동작하고 있었다.

## 6. 최종 판단

운영실 신규 등록은 이제:

- 일반 신청이 아니라
- 관리자 직행 입력

으로 동작한다.

즉:

- 결재 대기에 뜨지 않고
- DB에 바로 `approved`로 저장되며
- 메일도 보내지 않고
- `AdminRequestCreate` 로그만 남는다.
