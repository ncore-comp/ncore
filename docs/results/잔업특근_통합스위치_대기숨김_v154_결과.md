# 잔업/특근 통합 스위치 및 제출 대기 숨김 v154 결과
# Work Report Unified Toggle and Pending Section Hide v154 Result

## 요약
## Summary

- `잔업`과 `특근` 신청 기능을 하나의 정책으로 묶었다.
- Unified `overtime` and `holiday work` application controls into a single policy.

- `잔업/특근 신청 기능`이 꺼지면 홈 대시보드의 `잔업/특근 제출 대기`도 함께 숨기도록 바꿨다.
- When the `Work Report Application` feature is turned off, the home dashboard `Pending Overtime/Holiday Work Submission` section is now hidden as well.

## 반영 내용
## Applied Changes

1. `ui_permissions_v143_codex.js`
1. `ui_permissions_v143_codex.js`

- 설정 탭의 `잔업 신청 기능`, `특근 신청 기능` 두 스위치를 `잔업/특근 신청 기능` 하나로 통합했다.
- Replaced the separate `Overtime Application` and `Holiday Work Application` switches with a single `Work Report Application` switch.

- 저장 시 `featureOvertime`, `featureHolidayWork`를 같은 값으로 함께 저장한다.
- On save, `featureOvertime` and `featureHolidayWork` are now stored together with the same value.

2. `ui_dashboard_v154_codex.js`
2. `ui_dashboard_v154_codex.js`

- 홈의 `잔업/특근 제출 대기` 블록은 `app.isAnyWorkReportFeatureEnabled()`가 켜져 있을 때만 렌더되도록 바꿨다.
- The home `Pending Overtime/Holiday Work Submission` block now renders only when `app.isAnyWorkReportFeatureEnabled()` is enabled.

## 기대 결과
## Expected Result

- `권한/설정 > 잔업/특근 설정`에는 스위치가 1개만 보여야 한다.
- `Permissions/Settings > Work Report Settings` should now show only one switch.

- 해당 스위치를 끄면:
- When that switch is turned off:
  - 잔업/특근 신청 버튼이 사라지거나 비활성화되고
  - the work report application button disappears or becomes inaccessible
  - 홈의 `잔업/특근 제출 대기` 블록도 함께 숨겨진다.
  - and the home `Pending Overtime/Holiday Work Submission` block is hidden as well.
