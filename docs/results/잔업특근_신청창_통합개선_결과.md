# 잔업특근 신청창 통합개선 결과

## 작업 일시
- 2026-04-01

## 반영 내용
- 메인 화면의 `잔업 신청`, `특근 신청` 버튼을 제거하고 `잔업/특근신청` 버튼 1개로 통합했습니다.
- `잔업/특근신청` 버튼은 `잔업 신청 기능` 또는 `특근 신청 기능` 중 하나라도 켜져 있을 때만 보입니다.
- `보고 추가` 흐름은 더 이상 `연차 신청` 모달을 사용하지 않고, **전용 `잔업/특근 신청` 모달**을 열도록 변경했습니다.
- 기존 `연차 신청` 모달의 종류 목록에서 `잔업 보고`, `특근 보고`를 제거했습니다.

## 새 흐름
1. 메인 화면에서 `잔업/특근신청` 클릭
2. `정산월 잔업/특근 현황` 모달 열림
3. `보고 추가` 클릭
4. 전용 `잔업/특근 신청` 모달 열림
5. `구분(잔업/특근)` 선택 후 저장

## 전용 신청 모달 구성
- 구분
- 날짜
- 시작 시간
- 보고 시간
- 사유
- 업무내용
- 요청 부서
- 비고

## 동작 기준
- `잔업`
  - 저장된 퇴근시간 기준으로 시작시간 자동 적용
  - 평일 기본
  - 휴일에는 같은 날짜 특근 보고가 이미 있는 경우에만 추가 잔업 허용
- `특근`
  - 시작시간 직접 입력
  - 주말/공휴일에도 보고 가능
- 두 경우 모두 승인 절차 없이 `reported`로 저장

## 유지한 것
- 저장 구조는 기존 `requests` 기반 재사용
- 사용 필드:
  - `reportCategory`
  - `workDetail`
  - `requestDept`
  - `note`
  - `requestedStartAt`
  - `requestedEndAt`
  - `reportedHours`
