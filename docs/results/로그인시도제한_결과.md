# 로그인 시도 제한 결과

## 1. 목적

- 로그인 실패가 반복될 때 brute force 시도를 막는다.
- 기준은 `5회 실패 시 5분 차단`이다.

쉽게 말하면:
- 비밀번호를 계속 틀리면 경비실이 5분 동안 문을 잠그는 장치를 단 것이다.

## 2. 반영 내용

### 2.1 차단 기준

- 같은 `아이디 + 접속 IP`
- 로그인 실패 `5회`
- 이후 `5분` 차단

### 2.2 성공 로그인 시 처리

- 성공하면 해당 실패 기록 삭제
- 즉 차단 카운트가 다시 0부터 시작

## 5. 실제 검증 결과

### 5.1 차단 검증

테스트 아이디:
- `blocktestuser`

결과:
- 1~5회: `LOGIN_INVALID`
- 6회: `LOGIN_BLOCKED`
- `retryAfterSec: 300`

즉 설정한 `5회 실패 -> 5분 차단`이 정상 동작했다.

### 5.1-1 사용자 안내 문구 개선

- 로그인 실패 시:
  - `아이디 또는 비밀번호가 올바르지 않습니다. (1/5)`
  - `아이디 또는 비밀번호가 올바르지 않습니다. (2/5)`
  - 같은 형식으로 카운트 안내
- 차단 시:
  - `비밀번호를 5회 잘못 입력해 로그인할 수 없습니다. 약 5분 후 다시 시도해 주세요.`

즉 사용자는 지금 몇 번 실패했는지와, 차단되면 얼마 뒤에 다시 시도할 수 있는지 함께 알 수 있다.

### 5.2 성공 로그인 시 기록 초기화 검증

테스트 아이디:
- `Web관리자 (id: 0)`

흐름:
- 비밀번호 오입력 2회
- 비밀번호 `0`으로 정상 로그인

결과:
- 성공 로그인 후 `loginAttempts` 기록 삭제 확인

즉:
- 실패 기록은 남지만
- 정상 로그인하면 다시 초기화된다.

## 6. 사용자 체감 메시지

- 로그인 차단 시 프런트에는
  - `비밀번호 5회 오류로 5분간 로그인할 수 없습니다.`
  로 안내된다.
