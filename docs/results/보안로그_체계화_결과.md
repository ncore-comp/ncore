- 6회 실패: `LOGIN_BLOCKED`

비밀번호:

- 정상 변경: 성공
- `securityLogs`에 실패/성공 기록 저장 확인

권한:
- 관리자 사용자 저장 시 `USER_PRIVILEGED_UPDATE` 기록 확인

운영실:
- CSV 다운로드, 수동 휴일 저장/삭제가 각각 보안 로그로 기록되는 것 확인

## 7. 주의 사항

- 이번 1차는 `기록`까지만 완료
- 운영실에서 `securityLogs`를 조회하거나 CSV로 내리는 화면은 아직 없음
- 그 부분은 2차 과제로 분리

또한 검증 중 `테스트1(id=1)`의 비밀번호를 잠시 `0001`로 바꿨다가,
마지막에 다시 해시된 `0`으로 원복 완료
