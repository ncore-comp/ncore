# Master Permissions Stability Test and Analysis v127

## 1. 목적
## 1. Purpose

- 오늘 2026-04-07 오전부터 `권한/설정` 페이지에서 반복적으로 발생한 한글 깨짐과 기능 연결 끊김을 분리해서 확인한다.
- Separate today's repeated `master permissions` issues into text corruption problems and functional wiring problems.

## 2. 현재 상태 요약
## 2. Current State Summary

- 현재 `v127` 배포본에서 `권한/설정` 페이지의 한글 문자열은 광범위하게 깨져 있다.
- In the current `v127` deployment, Korean strings in the `master permissions` page are broadly corrupted.

- 기능 연결은 일부 복구되었지만, 페이지 전체를 신뢰할 수 있는 상태는 아니다.
- Some function wiring has been restored, but the page as a whole is not yet trustworthy.

- 특히 `권한/설정` 파일은 텍스트 손상과 함수 재연결이 섞여 있어서, 부분 수정이 다시 다른 문제를 만들 가능성이 높다.
- This page mixes text corruption with incremental function reconnects, so partial fixes are likely to create new regressions.

## 3. 핵심 분석
## 3. Key Analysis

### 3-1. 확인된 사실
### 3-1. Confirmed Facts

1. `ui_permissions_v127_codex.js`는 현재 한글이 깨져 있다.
1. `ui_permissions_v127_codex.js` is currently text-corrupted.

2. 우리가 정상 기준선으로 믿었던 `backups/2026-04-06_js-shared-phase4_v123/ui_permissions_v123_codex.js`도 이미 같은 형태로 깨져 있다.
2. The presumed clean baseline at `backups/2026-04-06_js-shared-phase4_v123/ui_permissions_v123_codex.js` is already corrupted in the same way.

3. 더 이른 버전인 `ui_logic_new_design_v114_codex.js`의 권한/설정 구간은 한글이 정상이다.
3. The earlier monolithic file `ui_logic_new_design_v114_codex.js` still contains clean Korean strings in the permissions section.

4. 즉, `v123 phase4 백업 복구`는 정상 복구가 아니라 `이미 깨진 분할본을 다시 덮어쓴 것`이었다.
4. That means the prior `restore from v123 phase4 backup` was not a true restoration; it reapplied an already corrupted split file.

### 3-2. 가장 유력한 원인
### 3-2. Most Likely Root Cause

- `권한/설정` 파일을 분리하거나 대량 치환하는 과정에서, UTF-8 기준 파일이 다른 인코딩으로 다시 저장되면서 문자열이 깨진 것으로 보인다.
- During extraction or large-scale replacement of the permissions file, a UTF-8 source appears to have been rewritten using a different encoding, corrupting the Korean literals.

- 이후 기능 복구 작업이 `이미 깨진 파일`을 기준으로 계속 이어져서, 텍스트 손상과 기능 손상이 같이 누적되었다.
- Later recovery work continued on top of an already corrupted file, so text damage and function damage accumulated together.

### 3-3. 현재 진단 결론
### 3-3. Current Diagnostic Conclusion

- 지금 문제는 단순한 라벨 수정이 아니다.
- The problem is not a simple label fix.

- `권한/설정` 페이지는 현재:
- The current `master permissions` page has:
  - 텍스트 기준선 손상
  - a corrupted text baseline
  - 부분 기능 재연결 이력
  - a history of partial function reconnects
  - 신뢰할 수 없는 복구 기준선
  - an unreliable restore baseline

- 따라서 다음 복구는 `깨진 파일을 부분 수정`하는 방식이 아니라, `정상 한글 기준선`을 다시 확보한 뒤 필요한 최신 기능만 재적용하는 방식이 더 안전하다.
- Therefore, the next repair should not keep patching the corrupted file. It should re-establish a clean Korean baseline first, then reapply only the needed latest behavior.
